## 발견된 문제점 (확인 vs 추정, 심각도)

### [확인] 자동화 테스트 부재 — 심각도: 높음
`tests/`가 비어 있고 pytest 수집 0건. 그러나 README.md(365행)와 `docs/TEST_REPORT_V2.md`는 "104건 통과"를 명시. 문서와 실제 코드베이스 간 불일치이며, 회귀 방지 장치가 전무하다.
- 증거: `ls tests/` → `__init__` 없음; `pytest --collect-only` → "no tests collected".

### [확인] requirements.txt와 실제 설치 버전의 대규모 불일치 — 심각도: 높음
requirements.txt가 명시한 버전 범위와 venv 실제 설치 버전이 메이저 단위로 어긋난다.
- langchain: 요구 `>=0.3.0` ↔ 설치 `1.1.0` (메이저 점프)
- crewai: 요구 `>=1.9.0` ↔ 설치 1.6.0/1.7.2 (요구 미달)
- fastapi: 설치 0.122.0/0.127.0
- 즉, requirements.txt로 깨끗한 재설치 시 현재 동작과 다른 버전이 깔릴 가능성이 높아 재현성이 보장되지 않는다 (추정: 클린 재설치 동작은 미검증).

### [확인] venv 내 중복 dist-info 68종 — 심각도: 중간
동일 패키지의 서로 다른 버전 메타데이터가 공존(예: crewai 1.6.0 & 1.7.2, fastapi 0.122.0 & 0.127.0).
- 증거: dist-info 중복 68건. `pip list`는 0.122.0/1.6.0을 보고하나 실제 `import`는 0.127.0/1.7.2를 로드 — pip 메타데이터와 런타임 해석이 불일치하는 오염된(dirty) 환경.
- 현재 임포트는 동작하나, 향후 의존성 충돌·예측 불가 동작의 원인이 될 수 있음.

### [확인] AutoGen 프레임워크 미동작 — 심각도: 중간
`src/agents/autogen_orchestrator.py`가 `from autogen_agentchat.agents import AssistantAgent`에서 `ModuleNotFoundError`. requirements.txt는 `autogen-agentchat>=0.7.0`(신 AG2)을 요구하나 venv에는 구 `autogen 0.10.3`만 설치됨.
- 영향: AutoGen 경로 라우트는 동작 불가. 단, `multi_agent_routes.py`에서 lazy import + try/except로 감싸 호출 시 HTTP 500을 반환하며 앱 전체는 크래시하지 않음(확인).

### [확인] DSPy / LlamaIndex 모듈 미설치 + 최상위 무방비 임포트 — 심각도: 중간
- `src/core/dspy_provider.py:13` `import dspy` → ModuleNotFoundError (dspy 미설치).
- `src/core/llamaindex_rag.py:17` `from llama_index.core import (...)` → ModuleNotFoundError (llama_index 미설치).
- 두 모듈 모두 최상위에서 무방비로 임포트하나, `advanced_framework_routes.py`가 핸들러 내부에서 lazy import + try/except로 호출하여 실패 시 HTTP 500을 반환(graceful). 앱 기동·핵심 기능에는 영향 없음(확인).
- 즉 README/CLAUDE.md가 광고하는 "7개 프레임워크" 중 DSPy·LlamaIndex·AutoGen 3개는 현재 환경에서 실동작 불가 상태.

### [확인] 폐기 예정(deprecated) API 사용 — 심각도: 낮음
- `main.py:131, 168` `@app.on_event("startup"/"shutdown")` → FastAPI DeprecationWarning(향후 lifespan 핸들러로 이관 필요).
- `config/settings.py`의 class-based `Config` → Pydantic V2 DeprecationWarning(ConfigDict 권장, V3에서 제거 예정).
- 현재는 경고만 발생, 동작에는 지장 없음(확인).

### [확인] CORS 전면 개방 — 심각도: 중간(보안)
`main.py`에서 `allow_origins=["*"]` 와 `allow_credentials=True` 를 동시 설정. 자격증명 허용 상태의 와일드카드 오리진은 권장되지 않는 구성이며, 운영 배포 시 보안 위험. CLAUDE.md가 강조하는 "온프레미스 보안 우선" 기조와 상충.

### [추정] 하드코딩된 절대경로 의존 — 심각도: 낮음~중간
`main.py` 및 CLAUDE.md 컨벤션이 `sys.path.append('/home/ubuntu-02/ai_project/100K-Expert')` 를 고정 사용. 다른 경로로 배포 시 임포트가 깨질 수 있음(추정 — 현재 경로에서는 정상).

### [확인] CI / Docker 부재 — 심각도: 낮음
`.github/`, Dockerfile, docker-compose, CI YAML 모두 없음(확인). 자동 빌드·배포·검증 파이프라인 미비.

## 조치한 내용

본 감사에서 **소스 코드 변경은 적용하지 않았다.** 사유는 다음과 같다.

- 미설치 의존성(dspy, llama_index, autogen-agentchat)은 환경 제약(대형 다운로드 금지) 및 무거운 전이 의존성 때문에 설치하지 않았다. 또한 라우트단의 lazy import + try/except가 이미 graceful degradation을 제공하여, 무방비 최상위 임포트를 try/except로 감싸는 변경은 실동작 개선 없이 표면만 바꾸는 비검증 변경에 해당한다.
- deprecated `on_event` → `lifespan` 이관, CORS 정책 수정, requirements.txt 버전 정정 등은 동작/배포 거동을 바꾸는 변경으로, 본 감사의 "저위험·검증 가능" 원칙을 벗어나며 충분한 검증 없이는 회귀 위험이 있어 적용하지 않고 아래 권고로 남긴다.

수행한 것은 read-only 점검과 일회성 임포트/TestClient 실행뿐이며, `data/`·`.git/`에 부수효과를 남기지 않았음을 확인하였다.

5. **venv/bin/pip 깨짐 (낮음)**: shebang이 구 경로(`/home/wdlab/...`)를 가리킴. `python -m pip` 우회는 가능하나, venv 재생성으로 근본 해결 권장.
6. **deprecated API 이관 (낮음)**: `on_event`→lifespan, Pydantic class-based Config→ConfigDict. 충분한 회귀 테스트와 함께 진행 권장.

## 종합 판단

- **런타임 기동 가능성: 동작 가능(확인).** venv(Python 3.12.3) 인터프리터로 `main` 임포트, 전 소스 바이트컴파일, 전 라우트 모듈 임포트, 핵심 엔드포인트 TestClient 200 응답이 모두 확인되었다. Ollama 인프라 및 설정 모델도 가용하다.
- 단, **3개 AI 프레임워크(DSPy/LlamaIndex/AutoGen)는 의존성 미설치로 실동작 불가**하며, 해당 경로는 graceful 500으로 격리된다(확인). LLM 실추론 동작은 본 감사에서 미검증(추정).
- **자동화 테스트가 전무**하고 문서는 사실과 다른 테스트 통과 수치를 주장한다(확인). 의존성 환경은 중복 메타데이터로 오염되어 재현성이 보장되지 않는다(확인).
- 결론: 데모/단일 환경에서의 코어 웹 기능은 기동·응답하나, (1) 검증 자산 부재, (2) 환경 재현성 결함, (3) 광고된 기능의 부분 미동작이라는 세 가지가 신뢰성 측면의 핵심 약점이다. 운영 신뢰성 확보 전 위 권고 1~4번의 선제 처리가 필요하다.
