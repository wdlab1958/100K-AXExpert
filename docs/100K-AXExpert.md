## 1. 플랫폼 개요

### 1.1 플랫폼 소개

100K-AX Expert Platform은 기업의 AI 전환(AX: AI Transformation)을 지원하는 전문 컨설팅 및 10만 AX 전문인력 양성 통합 플랫폼이다. 본 플랫폼은 Ollama 기반 로컬 LLM과 7개 멀티 에이전트 프레임워크를 활용하여 체계적인 AI 도입 컨설팅 서비스를 제공하며, 100K 목표 달성을 위한 AX 전문가 발굴·양성·인증 체계를 운영한다.

### 1.2 화면 구성

본 플랫폼은 단일 페이지 애플리케이션(SPA) 구조로 설계되어 있으며, 화면은 크게 세 영역으로 구분된다.

| 영역 | 위치 | 설명 |
|------|------|------|
| **사이드바** | 좌측 | 전체 메뉴 트리 및 Phase별 네비게이션. 드래그로 폭 조정이 가능하다. |
| **워크스페이스** | 중앙 | 선택한 메뉴에 대응하는 작업 영역. 폼, 차트, 테이블 등이 표시된다. |
| **헤더** | 상단 | 플랫폼 타이틀 및 현재 위치 표시 영역이다. |

### 1.3 사이드바 구조

사이드바는 Phase 단위로 구분되며, 각 Phase 내에 섹션(Section)이 존재하고, 섹션 내에 개별 메뉴 항목이 배치되어 있다. 섹션 헤더를 클릭하면 해당 섹션의 하위 메뉴를 접거나 펼칠 수 있다. 사이드바 좌측 경계에 위치한 리사이즈 핸들을 드래그하면 사이드바 폭을 자유롭게 조정할 수 있다.

### 1.4 기술 스택

| 분류 | 기술 |
|------|------|
| LLM | Ollama (llama3.2:3b, Local) |
| 멀티 에이전트 | LangGraph, CrewAI, AutoGen (AG2), DSPy, LangChain, LlamaIndex, Native |
| Backend | FastAPI + Uvicorn, Python 3.12 |
| Frontend | Jinja2 + HTML5, Bootstrap 5 Dark Theme (Glassmorphism), Chart.js |
| Database | SQLite (aiosqlite) |
| 총 API | 171개 |

## 2. START: 대시보드

**메뉴 경로**: START > 대시보드
**페이지 ID**: `page-dashboard`

대시보드는 플랫폼 접속 시 최초로 표시되는 메인 화면으로, 플랫폼 전반의 운영 현황을 한눈에 파악할 수 있도록 설계되었다.

### 2.1 주요 지표 카드

화면 상단에 4개의 통계 카드가 수평으로 배치되어 있다.

| 카드명 | 표시 항목 | 설명 |
|--------|----------|------|
| **전체 프로젝트** | 프로젝트 수 및 활성 프로젝트 수 | 등록된 컨설팅 프로젝트의 총 수를 표시한다. 활성 상태의 프로젝트 수가 하단에 별도로 표기된다. |
| **AI 성숙도** | Level 1~5 | 현재 선택된 프로젝트의 AI 성숙도 수준을 표시한다. CMMI 기반 5단계 모델을 기준으로 한다. |
| **예상 ROI** | 백분율(%) | AI 도입에 따른 예상 투자 수익률을 백분율로 표시한다. |
| **시스템 상태** | 정상/오류 | 플랫폼 구성 서비스(API, DB, Ollama)의 통합 상태를 표시한다. 마지막 점검 시각이 하단에 표기된다. |

### 2.2 컨설팅 진행 현황

5단계 컨설팅 프레임워크의 진행 상태를 시각적으로 표현하는 스테이지 프로그레스 바가 표시된다.

| 단계 | 명칭 | 클릭 시 동작 |
|------|------|-------------|
| Stage 1 | 전략 수립 | Stage 1 워크스페이스로 이동한다. |
| Stage 2 | 설계 정의 | Stage 2 워크스페이스로 이동한다. |
| Stage 3 | 구축 | Stage 3 워크스페이스로 이동한다. |
| Stage 4 | 확산 | Stage 4 워크스페이스로 이동한다. |
| Stage 5 | 운영 | Stage 5 워크스페이스로 이동한다. |

완료된 단계는 체크 아이콘으로 표시되며, 현재 진행 중인 단계는 활성 스타일로 강조된다.

### 2.3 서비스 통계

기간별 서비스 활동 통계를 조회할 수 있다. 상단의 기간 필터 버튼(오늘, 이번 주, 이번 달)을 클릭하여 조회 기간을 전환한다.

| 항목 | 설명 |
|------|------|
| **분석 건수** | 성숙도, 기회 발굴, 로드맵, 시나리오, 요건 정의, 아키텍처, 거버넌스 분석 수행 건수의 합계이다. |
| **보고서 생성** | 자동 생성된 컨설팅 보고서 수이다. |
| **AI 상담** | AI 전문가 상담 세션 수이다. |
| **리스크 레벨** | 현재 리스크 평가 수준을 표시한다. |
| **활성 컨설턴트** | 현재 활성 상태인 컨설턴트(에이전트) 수를 표시한다. |

### 2.4 연동 대시보드 상태

플랫폼의 핵심 구성 서비스에 대한 연결 상태를 실시간으로 모니터링한다.

| 서비스 | 설명 |
|--------|------|
| **Ollama LLM** | 로컬 LLM 서버(Ollama)의 연결 상태이다. 정상 시 녹색 상태 점이 표시된다. |
| **데이터베이스** | SQLite 데이터베이스의 연결 상태이다. 프로젝트 목록 API 호출로 확인한다. |
| **API 서버** | FastAPI 백엔드 서버의 헬스체크 상태이다. `/api/v1/health` 엔드포인트를 통해 확인한다. |

### 2.5 AX 전문가 양성 모니터링

대시보드 중앙부에 AX 전문가 양성 현황을 종합적으로 모니터링하는 전용 섹션이 배치되어 있다. 우측 상단의 "새로고침" 버튼을 클릭하면 최신 데이터로 갱신된다.

#### 2.5.1 100K 목표 진행률

10만 명 AX 전문인력 양성 목표 대비 현재 등록 인원의 달성률을 프로그레스 바로 시각화한다. 좌측에 현재 등록 인원 수가 표시되고, 우측에 달성률이 백분율로 표기된다.

#### 2.5.2 KPI 카드

4개의 핵심 성과 지표(KPI) 카드가 수평으로 배치되어 있다.

| 카드명 | 설명 |
|--------|------|
| **등록 사용자** | 플랫폼에 등록된 AX 전문인력 양성 대상자의 총 수이다. |
| **참여 기업** | AX 전환 프로그램에 참여하고 있는 기업의 수이다. |
| **수행 과제** | 완료된 AX 과제의 총 건수이다. |
| **과제 완료율** | 전체 할당 과제 대비 완료 과제의 비율이다. |

#### 2.5.3 인증 레벨별 분포

5단계 인증 체계(Master, Expert, Specialist, Practitioner, Beginner)의 인원 분포를 수평 프로그레스 바로 시각화한다. 각 레벨별 인원 수가 우측에 표기된다.

| 레벨 | 색상 | 설명 |
|------|------|------|
| Master | 보라색 | AX 전략 수립 및 기업 전환 총괄 역량을 보유한 최상위 등급이다. |
| Expert | 황색 | 복합 AX 프로젝트 리드 및 ROI 달성 실적을 보유한 등급이다. |
| Specialist | 청색 | 도메인별 AX 전문성을 보유하고 과제 10건 이상을 완료한 등급이다. |
| Practitioner | 녹색 | AX 과제 3건 이상을 수행하고 기본 도구를 활용할 수 있는 등급이다. |
| Beginner | 회색 | 기본 AI/AX 이해를 갖추고 입문 과제를 완료한 등급이다. |

#### 2.5.4 산업 도메인별 현황

7개 산업 도메인(제조, 금융, 공공, 물류, 의료, 교육, 국방)의 Best Practice 보유 건수를 도넛(Doughnut) 차트로 시각화한다. 차트의 우측에 범례가 표시된다.

#### 2.5.5 빠른 이동 버튼

AX 관련 세부 페이지로 직접 이동할 수 있는 4개의 바로가기 버튼이 제공된다.

| 버튼 | 이동 대상 |
|------|----------|
| AX 기회 발굴 | Phase 04 > AX 기회 발굴 워크스페이스 |
| 학습 진도 관리 | Phase 04 > 양성 현황 대시보드 |
| 인증 레벨 관리 | Phase 04 > 전문가 인증 |
| 기업 AX 현황 | Phase 04 > 기업 AX 성과 |

### 2.6 AI 성숙도 진단 차트

레이더(Radar) 차트로 현재 프로젝트의 AI 성숙도 진단 결과를 4대 핵심 영역(전략/비전, 조직/역량, 데이터/기술, 프로세스/거버넌스)별로 시각화한다. 프로젝트가 선택되지 않은 경우 빈 차트가 표시된다.

### 2.7 Use Case 우선순위 차트

현재 프로젝트에서 발굴된 Use Case의 우선순위 분석 결과를 차트로 시각화한다. 가치-실행 용이성 분석 결과를 기반으로 Quick Win, Strategic, Fill-in, Reconsider 4개 분류가 표현된다.

### 2.8 프로젝트 목록

등록된 모든 컨설팅 프로젝트를 테이블 형식으로 표시한다.

| 열 | 설명 |
|-----|------|
| 프로젝트명 | 프로젝트의 이름이다. 클릭 시 해당 프로젝트가 선택된다. |
| 산업 | 대상 기업의 산업 분류이다. |
| 진행률 | 컨설팅 전체 진행률을 백분율로 표시한다. |
| 상태 | 프로젝트의 현재 상태(진행 중, 완료 등)를 표시한다. |
| 최종 수정 | 마지막으로 데이터가 수정된 일시이다. |
| 작업 | 프로젝트 선택, 삭제 등의 액션 버튼이다. |

"새로고침" 버튼을 클릭하면 프로젝트 목록을 다시 불러오며, "새 프로젝트" 버튼을 클릭하면 새 프로젝트 생성 페이지로 이동한다.

### 2.9 워크플로우 진행 상태

우측 패널에 현재 선택된 프로젝트의 워크플로우 진행 상태가 표시된다.

#### 추가 분석 현황

| 인력 구성 | 필수 인력 구성 분석 완료 여부를 표시한다. |

| 최종 보고서 생성 | 컨설팅 결과를 종합한 최종 보고서를 생성한다. |

## 3. START: 새 프로젝트

**메뉴 경로**: START > 새 프로젝트
**페이지 ID**: `page-new-project`

새로운 컨설팅 프로젝트를 생성하기 위한 기업 정보 입력 폼이다. 모든 필수 항목을 입력한 후 "컨설팅 시작" 버튼을 클릭하면 프로젝트가 생성되고 Stage 1로 자동 이동한다.

### 3.1 기업 기본 정보

| 필드 | 유형 | 필수 | 설명 |
|------|------|------|------|
| 프로젝트명 | 텍스트 | 예 | 컨설팅 프로젝트의 명칭을 입력한다. |
| 기업명 | 텍스트 | 예 | 대상 기업의 공식 명칭을 입력한다. |
| 산업 분류 | 드롭다운 | 예 | 제조업, 금융업, 의료/헬스케어, 유통/물류, IT 서비스, 공공, 기타 중 선택한다. |
| 기업 규모 | 드롭다운 | 예 | 스타트업(50인 미만), 중소기업(50-300인), 중견기업(300-1000인), 대기업(1000인 이상) 중 선택한다. |

### 3.2 IT 인프라 현황

| 필드 | 유형 | 설명 |
|------|------|------|
| 클라우드 환경 보유 | 체크박스 | 체크 시 클라우드 제공자 입력 필드가 표시된다. |
| 클라우드 제공자 | 텍스트 | AWS, Azure, GCP, NCP 등을 콤마로 구분하여 입력한다. |
| 데이터 웨어하우스 보유 | 체크박스 | 데이터 웨어하우스 보유 여부이다. |
| 데이터 레이크 보유 | 체크박스 | 데이터 레이크 보유 여부이다. |
| 서버 수 | 숫자 | 보유 서버의 총 대수를 입력한다. 기본값은 10이다. |
| 레거시 시스템 수 | 숫자 | 운영 중인 레거시 시스템의 수를 입력한다. |
| GPU 서버 보유 | 체크박스 | GPU 서버 보유 여부이다. |
| API 연동 가능 여부 | 체크박스 | 외부 API 연동 가능 여부이다. |

#### 4.2.1 성숙도 5단계 탭

CMMI 기반 AI 성숙도 5단계 모델의 각 단계를 설명한다.

| 단계 | 명칭 | 설명 |
|------|------|------|
| Level 1 | Initial | AI에 대한 인식은 있으나 체계적 접근이 부재한 초기 단계이다. |
| Level 2 | Repeatable | 일부 영역에서 반복적으로 AI를 활용하기 시작한 단계이다. |
| Level 3 | Defined | 조직 차원의 AI 전략과 프로세스가 정의된 단계이다. |
| Level 4 | Managed | AI가 비즈니스 프로세스에 통합되어 정량적으로 관리되는 단계이다. |
| Level 5 | Optimized | AI가 지속적으로 최적화되며 혁신을 주도하는 단계이다. |

#### 4.2.2 진단 영역 탭

| 데이터 및 기술 | 데이터 인프라 및 접근성, 데이터 품질 및 거버넌스, MLOps 플랫폼 활용 수준, 클라우드 활용 및 확장성 |
| 프로세스 및 거버넌스 | 개발 방법론 표준화, 모델 검증 및 배포 승인 절차, 윤리 및 리스크 관리, 모니터링 및 운영 체계 |

#### 4.2.4 성숙도 평가 탭

실제 성숙도 평가를 수행하는 폼이 제공된다.

**평가 항목 구조**: 4대 영역 × 4개 세부 항목 = 총 16개 항목을 1~5점 슬라이더로 평가한다.

**목표 성숙도 수준**: 각 영역별로 목표 Level(2~5)을 드롭다운에서 선택한다.

**Gap 분석**: "Gap 분석 생성" 버튼을 클릭하면 현재 수준과 목표 수준 간의 차이를 자동으로 분석하여 결과를 표시한다.

"성숙도 평가 저장" 버튼을 클릭하면 평가 결과가 현재 프로젝트에 저장된다.

### 4.3 가치-실행 매핑

**메뉴 경로**: Phase 01 > 컨설팅 방법론 > 가치-실행 매핑
**페이지 ID**: `page-value-mapping`

AI 도입 과제의 비즈니스 가치와 실행 용이성을 평가하여 우선순위를 결정하는 페이지이다. 4개의 탭으로 구성되어 있다.

#### 4.3.1 평가 기준 탭

비즈니스 가치와 실행 용이성을 평가하기 위한 기준 항목을 테이블로 제시한다.

**비즈니스 가치 기준**: 예상 ROI, 비용 절감 효과, 매출 증대 기여도, 전략적 중요도, 고객 만족도 향상

**실행 용이성 기준**: 기술적 복잡성, 데이터 가용성, 필요 자원, 조직 변화 관리, 시스템 통합

#### 4.3.2 우선순위 분류 탭

2×2 매트릭스의 4개 사분면을 설명한다.

| 사분면 | 조건 | 권장 기간 |
|--------|------|----------|
| Quick Win | 높은 가치 + 높은 용이성 | 6개월 이내 |
| Strategic | 높은 가치 + 낮은 용이성 | 12~24개월 |
| Fill-in | 낮은 가치 + 높은 용이성 | 여유 자원 활용 |
| Reconsider | 낮은 가치 + 낮은 용이성 | 보류 또는 재검토 |

#### 4.3.3 과제 평가 탭

개별 과제를 등록하고 평가하는 폼이 제공된다.

**입력 항목**: 과제명, 비즈니스 영역(드롭다운: 영업/마케팅, 운영/제조, 고객 서비스, 재무/회계, 인사/조직, 기타), 과제 설명

**비즈니스 가치 평가**: 5개 항목을 1~5점 슬라이더로 평가한다. 합계 점수(25점 만점)가 자동 계산된다.

**실행 용이성 평가**: 5개 항목을 1~5점 슬라이더로 평가한다. 일부 항목은 역순으로 채점된다. 합계 점수(25점 만점)가 자동 계산된다.

평가 완료 후 자동으로 사분면 분류와 예상 소요 기간이 표시된다. "매트릭스에 추가" 버튼을 클릭하면 2×2 매트릭스에 과제가 배치된다.

#### 4.3.4 2×2 매트릭스 탭

등록된 모든 과제가 2×2 매트릭스 시각화 영역에 배치된다. X축은 비즈니스 가치, Y축은 실행 용이성을 나타낸다. 우측에 각 사분면별 과제 수 통계가 표시된다. "매트릭스 저장" 버튼으로 현재 상태를 저장할 수 있다.

### 6.1 프레임워크 요약 카드

화면 상단에 7개 프레임워크의 미니 상태 카드가 수평으로 배치되어 있다. 각 카드에는 프레임워크 아이콘, 상태 표시점(녹색=정상, 회색=미확인), 에이전트/모듈 수가 표시된다. 카드를 클릭하면 해당 프레임워크의 상세 카드로 자동 스크롤된다.

| 프레임워크 | 유형 | 에이전트/모듈 수 |
|-----------|------|----------------|

| CrewAI | Role-Based | 5 에이전트, 6 태스크 |
| AutoGen (AG2) | GroupChat | 5 에이전트 |
| DSPy | Signatures | 5 Signature, 5 CoT Module |
| LangChain | LCEL Pipeline | 5 Chain |
| LlamaIndex | RAG Workflow | 4 ISO 표준(15문서) |
| Native | Sequential | 5 컨설팅 에이전트 |

### 6.2 상세 카드 그리드

각 프레임워크에 대한 상세 정보 카드가 2열 그리드로 배치되어 있다. 각 카드에는 다음 정보가 포함된다.

- 프레임워크 명칭 및 유형
- 기능 태그(Feature Tags)
- 에이전트/모듈 목록
- "상태 조회" 버튼: 해당 프레임워크의 Health API를 호출하여 실시간 상태를 확인한다.
- "실행 테스트" 버튼: POST 요청으로 해당 프레임워크의 실행 테스트를 수행한다.

### 6.3 에이전트 협업 구조도

SVG 기반의 3계층 아키텍처 다이어그램이 표시된다.

```
상위 계층: 100K-AX Expert Orchestrator
중간 계층: 7개 Framework 노드
하위 계층: Ollama LLM (llama3.2:3b)
```

각 노드 간 연결선과 상태 표시 점이 포함되어 있으며, 그래디언트 컬러로 시각적 구분이 이루어진다.

### 6.4 상태 조회

화면 상단의 "상태 조회" 버튼을 클릭하면 3개의 Health API(`/api/v1/multi-agent/health`, `/api/v1/advanced-frameworks/health`, `/api/v1/agents/status`)를 병렬로 호출하여 전체 프레임워크의 상태를 한번에 갱신한다.

"결과 분석" 버튼을 클릭하면 전체 성숙도 점수, 성숙도 수준, 우선 개선 영역이 자동 계산되어 표시된다. "설문 저장" 버튼으로 결과를 저장할 수 있다.

## 10. Phase 03: ISO 38500 IT 거버넌스

### 10.1 IT 거버넌스 대시보드

**페이지 ID**: `page-it-governance-dashboard`

ISO 38500 IT 거버넌스의 종합 현황을 조회하는 대시보드이다.

**EDM 사이클 개요**: Evaluate(평가), Direct(지휘), Monitor(모니터링) 3개 활동의 점수와 진행률을 카드로 표시한다.

**6대 원칙 준수 현황**: 책임, 전략, 획득, 성과, 적합성, 인간 행동의 6가지 원칙별 준수율을 프로그레스 바로 시각화한다.

**ISO 38500 매핑 테이블**: 9개 영역에 대해 ISO 38500 활동과 플랫폼 모듈의 매핑을 제시한다.

**빠른 액세스 버튼**: EDM 사이클, 6대 원칙, 경영진 대시보드, 거버넌스 성숙도로 바로 이동할 수 있다.

### 10.2 EDM 사이클 관리

**페이지 ID**: `page-edm-cycle`

Evaluate-Direct-Monitor 거버넌스 사이클을 관리한다.

**사이클 다이어그램**: E→D→M→피드백 루프의 시각적 표현이 제공된다.

**활동 체크리스트**: 3개 영역(평가, 지휘, 모니터링) × 6개 활동 = 총 18개 체크항목을 관리한다.

| 영역 | 활동 |
|------|------|
| 평가(Evaluate) | 전략적 정렬 평가, 기술적 타당성 평가, 위험 평가, 역량 평가, 이해관계자 분석, ROI 추정 |
| 지휘(Direct) | AI 정책 수립, 전략/로드맵 승인, 예산 배정, 역할/책임 할당, 위험 대응 지시, 우선순위 결정 |
| 모니터링(Monitor) | 성과 모니터링, 준수 모니터링, 위험 모니터링, 품질 모니터링, 변화 모니터링, 사용자 만족도 추적 |

"진행률 계산" 및 "EDM 활동 저장" 기능을 제공한다.

### 10.3 6대 원칙 체크리스트

**페이지 ID**: `page-six-principles`

IT 거버넌스 6대 원칙별 체크리스트이다. 6개 원칙 × 6개 항목 = 총 36개 체크항목을 관리한다.

| 원칙 | 색상 | 체크 항목(6개) |
|------|------|---------------|
| 책임(Responsibility) | 청색 | 책임 정의, 거버넌스 체계 소유자, 윤리 위원회, 역할/책임, 데이터 관리, RACI 매트릭스 |
| 전략(Strategy) | 녹색 | 비즈니스 전략 정렬, 문서화, KPI 정의, 우선순위 관리, 역량 개발, 투자 계획 |
| 획득(Acquisition) | 시안 | Build vs Buy 분석, TCO/ROI 분석, 벤더 평가, 위험 평가, 계약 SLA, 조달 프로세스 |
| 성과(Performance) | 황색 | KPI 정의/측정, SLA 정의/모니터링, ROI 추적, 성과 모니터링, 사용자 만족도, 성과 기반 개선 |
| 적합성(Conformance) | 적색 | 규제 식별, 개인정보 준수, AI 규제 모니터링, 정책 준수, 감사, ISO 표준 적용 |
| 인간 행동(Human Behaviour) | 회색 | 이해관계자 영향 평가, 변화 관리, 인간-AI 협업, 교육/훈련, 고충 채널, 인간 통제 보장 |

"준수율 계산" 및 "체크리스트 저장" 기능을 제공한다.

### 10.4 RACI 매트릭스

**페이지 ID**: `page-raci-matrix`

AI 거버넌스 활동에 대한 역할과 책임을 정의한다. R(실행), A(책임), C(자문), I(통보)를 활동별로 지정한다.

| 도메인 | 7개 산업 도메인 중 전문 도메인을 선택한다. |

#### 필터 및 검색

기업 ID와 부서명으로 실무자 목록을 필터링할 수 있다.

#### 실무자 목록 테이블

| 열 | 설명 |
|-----|------|
| 실무자 ID | 고유 식별자이다. |
| 이름 | 실무자의 성명이다. |
| 기업 | 소속 기업명이다. |
| 부서 | 소속 부서명이다. |
| 인증 등급 | 현재 인증 레벨(Beginner~Master)이다. |
| 완료 과제 | 완료한 AX 과제의 건수이다. |
| 액션 | "상세" 버튼을 클릭하면 상세 정보를 조회한다. |

#### 실무자 상세 정보

실무자를 선택하면 다음의 상세 정보가 표시된다.

| 항목 | 설명 |
|------|------|
| 인증 등급 | 현재 인증 레벨과 등급 색상이 표시된다. |
| 역량 점수 | 종합 역량 점수이다. |
| 다음 등급까지 | 다음 인증 레벨까지의 Gap 점수이다. |
| 과제 이력 | 할당된 과제의 목록과 상태(대기/진행/완료)가 표시된다. |
| 획득 뱃지 | 획득한 뱃지(인증 마크)가 표시된다. |

### 12.3 전문가 인증

**메뉴 경로**: Phase 04 > 학습 진도 관리 > 전문가 인증
**페이지 ID**: `page-certification`

AX 전문가 5단계 인증 체계를 관리하는 페이지이다.

#### 인증 레벨 카드

5단계 인증 레벨(Beginner, Practitioner, Specialist, Expert, Master)의 정의와 요건이 카드 형태로 표시된다. 페이지 진입 시 `/api/v1/training/certification/levels` API를 호출하여 각 레벨의 상세 요건을 로드한다.

#### 실무자 인증 현황 조회

실무자 ID를 입력하고 검색 버튼을 클릭하면 해당 실무자의 인증 현황이 표시된다.

| 항목 | 설명 |
|------|------|
| 현재 레벨 | 현재 인증 레벨과 설명이 표시된다. |
| 획득 뱃지 | 보유 뱃지(인증 마크) 목록이 표시된다. |
| Gap 분석 | 다음 레벨 달성을 위해 필요한 요건과 현재 부족한 항목이 분석되어 표시된다. |

### 12.4 기업 AX 성과

**메뉴 경로**: Phase 04 > 학습 진도 관리 > 기업 AX 성과
**페이지 ID**: `page-enterprise-ax`

기업 단위의 AX 전환 성과를 종합적으로 분석하는 페이지이다.

#### 기업 조회

#### KPI 카드(4개)

| 카드 | 설명 |
|------|------|

| 완료 과제 | 기업 내 완료된 AX 과제 총 건수이다. |
| ROI 달성 과제 | ROI 목표를 달성한 과제의 건수이다. |
| AX 전환율 | 기업의 전체 AX 전환 진행률이다. |

#### 부서별 AX 현황 테이블

| 열 | 설명 |
|-----|------|
| 부서 | 부서 명칭이다. |
| 인원 | 해당 부서의 등록 인원 수이다. |
| 완료 과제 | 해당 부서에서 완료한 과제 수이다. |
| ROI 달성 | ROI 목표를 달성한 과제 수이다. |
| 평균 등급 | 부서 내 평균 인증 레벨이다. |
| 전환율 | 부서의 AX 전환 진행률이다. |

#### 인증 등급 분포

기업 내 AX 전문인력의 인증 레벨별 분포를 시각적으로 표시한다.

## 15. 시스템 관리

### 15.1 AI 에이전트

**페이지 ID**: `page-agents`

7개 멀티 에이전트 프레임워크의 상태를 모니터링하고 테스트한다. (6장 참조)

### 15.2 보안 설정

**메뉴 경로**: 시스템 관리 > 보안 설정
**페이지 ID**: `page-security`

플랫폼의 보안 설정을 관리하는 페이지이다. 4개의 탭으로 구성되어 있다.

#### 탭 1: 온라인 LLM 제공자

외부 LLM 제공자를 등록하고 관리한다. 좌측에 등록된 제공자 목록이 표시되며, 우측에 지원되는 6개 제공자(Claude, ChatGPT, Gemini, Cohere, Perplexity, Azure OpenAI)가 안내된다.

"제공자 추가" 버튼을 클릭하면 등록 모달이 표시된다.

#### 탭 2: 데이터 분류

텍스트의 민감도를 자동 분류하는 기능이다. 테스트할 텍스트를 입력하고 "분류 테스트" 버튼을 클릭하면 분류 결과가 표시된다.

| 민감도 수준 | 설명 |
|------------|------|
| PUBLIC | 공개 가능한 데이터이다. |
| INTERNAL | 내부용 데이터이다. |
| CONFIDENTIAL | 기밀 데이터이다. |

#### 탭 3: 감사 로그

시스템 감사 로그를 조회한다. 감사 통계, 보안 알림, 상세 로그를 확인할 수 있다. 일간/주간/월간 보고서 생성 기능을 제공한다.

#### 탭 4: 질의 템플릿

AI 상담에 사용되는 질의 템플릿을 카테고리별로 관리한다.

### 15.3 운영 가이드

**페이지 ID**: `page-operations`

플랫폼 운영에 필요한 가이드 정보를 제공한다. 모니터링 데이터 조회 및 체크리스트 관리 기능이 포함되어 있다.
