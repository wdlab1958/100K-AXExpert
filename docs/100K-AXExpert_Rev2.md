## 1. 플랫폼 개요와 핵심 개념

### 1.1 플랫폼이 하는 일

100K-AX Expert Platform은 **"각 산업 도메인의 업무 전문가 10만 명을 AX 전문가로 양성한다"**는 목표를 가진 통합 플랫폼이다. 여기서 "AX 전문가"란 AI 기술을 직접 개발하는 엔지니어가 아니라, **자기 업무에 AI를 접목하여 비용을 절감하고 품질을 높이는 실무 전문가**를 의미한다.

📖 **용어 해설**

| 용어 | 일상적 의미 | 플랫폼에서의 의미 |
|------|------------|-----------------|
| **AI (Artificial Intelligence)** | 인공지능. 사람처럼 판단하는 컴퓨터 프로그램 | 업무 데이터를 학습하여 예측·분류·최적화를 수행하는 소프트웨어 |
| **ML (Machine Learning)** | 기계학습. AI를 만드는 구체적 기술 방법 | 과거 데이터에서 패턴을 학습하여 새 데이터에 대해 자동으로 판단하는 기술 |
| **AX (AI Transformation)** | AI 전환. 기존 업무를 AI 기반으로 바꾸는 것 | 수기/엑셀 기반 업무를 AI가 보조하거나 자율 운영하는 형태로 전환하는 과정 |
| **DX (Digital Transformation)** | 디지털 전환. 아날로그를 디지털로 바꾸는 것 | 종이 문서를 전산화하고, 수작업을 시스템으로 옮기는 과정. AX의 전 단계이다. |
| **ROI (Return on Investment)** | 투자 수익률 | AI 도입에 투자한 비용 대비 절감·매출 증가 효과의 비율 (%) |
| **LLM (Large Language Model)** | 대규모 언어 모델 | ChatGPT와 유사한 대화형 AI. 본 플랫폼에서는 Ollama 기반 로컬 LLM을 사용한다. |
| **SPA (Single Page Application)** | 단일 페이지 애플리케이션 | 페이지 전환 없이 하나의 화면에서 모든 기능을 사용하는 웹 앱 구조 |

## 2. DX-to-AX 전환 개념과 산업별 실무 사례

이 장은 본 플랫폼의 핵심 철학인 **DX-to-AX 전환**의 개념을 AI 지식이 없는 실무자의 관점에서 상세히 설명한다.

### 2.1 DX-to-AX 전환이란?

모든 업무는 다음 5단계의 성숙도 레벨을 거쳐 AI 기반 자율 운영으로 진화한다.

| DX-to-AX 레벨 | 명칭 | 일상적 설명 | 실무 예시 (품질 검사 업무) |
|:---:|------|------------|--------------------------|
| **Level 0** | Manual (수기 작업) | 종이, 수기 기록, 경험에 의존하는 단계 | 검사원이 눈으로 제품 외관을 검사하고 종이 체크리스트에 기록한다 |
| **Level 1** | Digital (전산화) | 시스템 도입, 데이터가 전자화된 단계 | ERP에 검사 결과를 입력하고, 엑셀로 불량률을 집계한다 |
| **Level 2** | Data-Driven (데이터 기반) | 축적된 데이터로 의사결정하는 단계 | 불량 데이터를 분석하여 "A 라인의 오전 10시 불량률이 높다"는 패턴을 발견한다 |
| **Level 3** | AI-Augmented (AI 보조) | AI가 사람을 보조하는 단계 | AI 비전 카메라가 불량을 자동 판별하고, 검사원이 AI 판정을 확인한다 |
| **Level 4** | Autonomous (AI 자율) | AI가 자율적으로 판단·조치하는 단계 | AI가 불량을 판별하고, 즉시 공정 파라미터를 자동 조정하여 불량을 예방한다 |

💡 **핵심 메시지**: AX는 "사람을 대체하는 것"이 아니라, "사람이 더 가치 있는 일에 집중할 수 있도록 반복적이고 오류가 발생하기 쉬운 업무를 AI에게 맡기는 것"이다.

### 2.2 산업 도메인별 DX-to-AX 전환 사례

본 플랫폼은 7개 산업 도메인을 지원한다. 각 도메인별로 실무자가 체감할 수 있는 구체적 전환 사례를 제시한다.

#### 2.2.1 제조업 (Manufacturing)

📖 **용어 해설**

| 용어 | 설명 |
|------|------|
| **예지보전 (Predictive Maintenance)** | 설비가 고장 나기 전에 센서 데이터를 AI가 분석하여 "곧 고장 날 것"을 미리 예측하는 기술이다. |
| **SPC (Statistical Process Control)** | 통계적 공정 관리. 공정의 품질 데이터를 통계적으로 분석하여 이상을 감지하는 방법이다. |
| **비전 검사 (Vision Inspection)** | 카메라로 제품 외관을 촬영하고, AI가 이미지를 분석하여 불량 여부를 자동 판별하는 기술이다. |
| **강화학습 (Reinforcement Learning)** | AI가 시행착오를 통해 최적의 행동을 스스로 학습하는 방식이다. 공정 조건 최적화에 활용된다. |

**관련 규제/표준**: ISO 9001, IATF 16949, HACCP, ISO 14001, ISO 45001

### 2.3 AX 적합도 점수 체계

플랫폼의 "AX 기회 발굴" 기능에서 각 기회의 적합도를 평가할 때 사용되는 4대 점수 체계이다.

📖 **용어 해설**

| 점수 | 가중치 | 의미 | 높은 점수를 받는 조건 |
|------|--------|------|---------------------|
| **ROI 잠재력 (roi_potential_score)** | 30% | 투자 대비 수익이 얼마나 클 것인가 | 현재 비용이 높고, 오류율이 높으며, 투입 인력이 많은 업무 |
| **구현 용이성 (implementation_ease_score)** | 25% | 얼마나 쉽게 구현할 수 있는가 | 난이도가 낮고(low), 데이터가 이미 수집되고 있는 업무 |
| **데이터 준비도 (data_readiness_score)** | 25% | AI 학습에 필요한 데이터가 준비되어 있는가 | 데이터가 이미 수집되고 있고, 데이터 양이 충분한 업무 |
| **조직 준비도 (org_readiness_score)** | 20% | 조직이 변화를 수용할 준비가 되어 있는가 | 투입 인력이 3명 이상이고, 업무 빈도가 일일/시간 단위인 경우 |

**종합 점수 계산**: `종합 = ROI잠재력 × 0.30 + 구현용이성 × 0.25 + 데이터준비도 × 0.25 + 조직준비도 × 0.20`

| **전체 프로젝트** | 프로젝트 수 및 활성 프로젝트 수 | 등록된 컨설팅 프로젝트의 총 수를 표시한다. 하단에 활성(진행 중) 상태의 프로젝트 수가 별도로 표기된다. "활성"이란 현재 컨설팅이 진행되고 있는 프로젝트를 의미한다. |
| **AI 성숙도** | Level 1~5 | 현재 선택된 프로젝트의 AI 성숙도 수준을 표시한다. CMMI 기반 5단계 모델을 기준으로 하며, Level 1(초기)에서 Level 5(최적화)까지 구분된다. 프로젝트가 선택되지 않은 경우 "-"로 표시된다. |
| **예상 ROI** | 백분율(%) | AI 도입에 따른 예상 투자 수익률을 백분율로 표시한다. 예를 들어 "150%"이면 1억 투자 시 1.5억의 효과가 기대된다는 의미이다. |
| **시스템 상태** | 정상/오류 | 플랫폼 구성 서비스(API 서버, DB, Ollama LLM)의 통합 상태를 표시한다. 모든 서비스가 정상이면 "정상", 하나라도 오류가 있으면 "오류"로 표시된다. |

### 3.2 컨설팅 진행 현황

5단계 컨설팅 프레임워크의 진행 상태를 시각적으로 표현하는 스테이지 프로그레스 바가 표시된다.

📖 **용어 해설**

| 용어 | 설명 |
|------|------|
| **5단계 컨설팅 프레임워크** | AI 도입 컨설팅을 체계적으로 수행하기 위한 단계별 절차이다. 전략 수립 → 설계 정의 → 구축 → 확산 → 운영의 5단계로 구성된다. |
| **스테이지 (Stage)** | 컨설팅 프레임워크의 각 단계를 의미한다. Stage 1부터 Stage 5까지 순차적으로 진행된다. |

| 단계 | 명칭 | 수행 내용 | 클릭 시 동작 |
|------|------|----------|-------------|
| Stage 1 | 전략 수립 | AI 성숙도 진단, 기회 발굴, 로드맵 수립 | Stage 1 워크스페이스로 이동한다 |
| Stage 2 | 설계 정의 | 데이터 전략, 아키텍처 설계, 기술 스택 선정 | Stage 2 워크스페이스로 이동한다 |
| Stage 3 | 구축 | 요건 정의, PoC 개발, 파일럿 구축 | Stage 3 워크스페이스로 이동한다 |
| Stage 4 | 확산 | 전사 확산, 변화 관리, 교육 훈련 | Stage 4 워크스페이스로 이동한다 |
| Stage 5 | 운영 | 지속적 모니터링, 성과 관리, 개선 | Stage 5 워크스페이스로 이동한다 |

완료된 단계는 체크 아이콘(✓)으로 표시되며, 현재 진행 중인 단계는 활성 스타일(밝은 색상)로 강조된다.

| **활성 컨설턴트** | 현재 활성 상태인 컨설턴트(AI 에이전트)의 수를 표시한다. |

### 3.5 AX 전문가 양성 모니터링

대시보드 중앙부에 AX 전문가 양성 현황을 종합적으로 모니터링하는 전용 섹션이 배치되어 있다. 우측 상단의 "새로고침" 버튼을 클릭하면 최신 데이터로 갱신된다.

📖 **용어 해설**

| 용어 | 설명 |
|------|------|
| **100K 목표** | 10만 명의 AX 전문인력을 양성한다는 플랫폼의 최종 목표이다. |
| **인증 레벨** | AX 전문가의 역량을 5단계(Beginner → Practitioner → Specialist → Expert → Master)로 구분한 등급 체계이다. |
| **도메인 (Domain)** | 산업 분야를 의미한다. 제조, 금융, 공공, 물류, 의료, 교육, 국방의 7개 도메인을 지원한다. |

#### 3.5.1 100K 목표 진행률

10만 명 AX 전문인력 양성 목표 대비 현재 등록 인원의 달성률을 프로그레스 바로 시각화한다. 좌측에 현재 등록 인원 수가 표시되고, 우측에 달성률이 백분율로 표기된다.

💡 **읽는 방법**: 프로그레스 바가 3.5%를 표시하면, 현재 3,500명이 등록되어 100,000명 목표의 3.5%를 달성했다는 의미이다.

#### 3.5.2 KPI 카드

4개의 핵심 성과 지표(KPI) 카드가 수평으로 배치되어 있다.

| 카드명 | 데이터 출처 | 상세 설명 |
|--------|-----------|----------|
| **등록 사용자** | `total_users` | 플랫폼에 등록된 AX 전문인력 양성 대상자의 총 수이다. 모든 산업 도메인·기업·부서의 실무자를 합산한다. |
| **참여 기업** | `total_companies` | AX 전환 프로그램에 참여하고 있는 기업의 수이다. 동일 기업의 여러 부서는 1개 기업으로 집계된다. |
| **수행 과제** | `total_tasks_completed` | 완료된 AX 과제의 총 건수이다. "완료"란 과제 상태가 `completed` 또는 `roi_achieved`인 경우를 포함한다. |
| **과제 완료율** | `completion_rate` | 전체 할당 과제 대비 완료 과제의 비율이다. 예를 들어 100건 중 30건이 완료되면 30%로 표시된다. |

#### 3.5.3 인증 레벨별 분포

5단계 인증 체계의 인원 분포를 수평 프로그레스 바로 시각화한다. 각 레벨별 인원 수가 우측에 표기된다.

| 레벨 | 색상 | 역할 정의 | 상세 설명 |
|------|------|----------|----------|
| **Master** | 보라색 | 산업 도메인 AX 전문가 | AX 전략 수립 및 기업 전환 총괄 역량을 보유한 최상위 등급이다. 완료 과제 100건 이상, ROI 달성 50건 이상, 부서 전환율 50% 이상, 외부 컨설팅 경험을 모두 갖추어야 한다. |
| **Expert** | 황색 | 기업 전사 AX 전략 수립 | 복합 AX 프로젝트 리드 및 ROI 달성 실적을 보유한 등급이다. 완료 과제 100건 이상, ROI 달성 50건 이상, 부서 전환율 50% 이상이 요구된다. |
| **Specialist** | 청색 | 부서 내 AX 리더 | 도메인별 AX 전문성을 보유하고 과제 50건 이상, ROI 달성 25건 이상, 부서 전환율 30% 이상을 달성한 등급이다. |
| **Practitioner** | 녹색 | 독립적 AX 과제 수행 | AX 과제 20건 이상, ROI 달성 10건 이상을 완료하고 기본 도구를 독립적으로 활용할 수 있는 등급이다. |
| **Beginner** | 회색 | AX 개념 이해 | 기본 AI/AX 개념을 이해하고 입문 과제 5건 이상을 완료한 등급이다. |

#### 3.5.4 산업 도메인별 현황

7개 산업 도메인(제조, 금융, 공공, 물류, 의료, 교육, 국방)의 Best Practice 보유 건수를 도넛(Doughnut) 차트로 시각화한다.

📖 **용어 해설**

| 용어 | 설명 |
|------|------|
| **Best Practice** | 특정 산업에서 성공적으로 검증된 AX 적용 사례이다. 예를 들어 제조업의 "AI 비전 검사"는 대표적인 Best Practice이다. |
| **도넛 차트 (Doughnut Chart)** | 원형 차트의 중심이 비어 있는 형태로, 각 항목의 비율을 시각적으로 비교할 수 있다. |

#### 3.5.5 빠른 이동 버튼

AX 관련 세부 페이지로 직접 이동할 수 있는 4개의 바로가기 버튼이 제공된다.

| 버튼 | 이동 대상 | 용도 설명 |
|------|----------|----------|
| AX 기회 발굴 | Phase 04 > AX 기회 발굴 | 자신의 업무 프로세스를 입력하여 AX 전환 기회를 자동 발굴한다 |
| 학습 진도 관리 | Phase 04 > 양성 현황 대시보드 | 10만 명 목표 대비 양성 현황을 확인한다 |
| 인증 레벨 관리 | Phase 04 > 전문가 인증 | 5등급 인증 체계와 등급별 요건을 확인한다 |
| 기업 AX 현황 | Phase 04 > 기업 AX 성과 | 기업 단위 AX 전환 성과를 종합 분석한다 |

### 3.6 AI 성숙도 진단 차트

📖 **용어 해설**

| 용어 | 설명 |
|------|------|
| **레이더 차트 (Radar Chart)** | 거미줄 형태의 차트로, 여러 항목의 수준을 동시에 비교할 수 있다. 바깥쪽으로 갈수록 점수가 높다. |

레이더 차트로 현재 프로젝트의 AI 성숙도 진단 결과를 4대 핵심 영역별로 시각화한다.

| 영역 | 평가 내용 |
|------|----------|
| **전략/비전** | 경영진의 AI 도입 의지, AI 전략 수립 여부, 로드맵 존재 여부를 평가한다 |
| **조직/역량** | AI 전담 조직 유무, AI 인력 보유 현황, 교육 프로그램 운영 여부를 평가한다 |
| **데이터/기술** | 데이터 수집·관리 체계, 분석 인프라, AI/ML 플랫폼 보유 여부를 평가한다 |
| **프로세스/거버넌스** | AI 개발·운영 프로세스 정립 여부, 윤리·보안 거버넌스 체계를 평가한다 |

### 3.7 Use Case 우선순위 차트

📖 **용어 해설**

| 용어 | 설명 |
|------|------|
| **Quick Win** | 구현이 쉽고 가치도 높은 과제. 가장 먼저 수행해야 하는 과제이다. |
| **Strategic** | 구현은 어렵지만 가치가 매우 높은 과제. 중장기 전략으로 추진해야 한다. |
| **Fill-in** | 구현은 쉽지만 가치가 상대적으로 낮은 과제. 여유가 있을 때 수행한다. |
| **Reconsider** | 구현도 어렵고 가치도 낮은 과제. 재검토가 필요한 과제이다. |

현재 프로젝트에서 발굴된 Use Case의 우선순위 분석 결과를 가치-실행 용이성 매트릭스 기반 4분면 차트로 시각화한다.

**빠른 실행 버튼**:

| 버튼 | 기능 설명 |

| **전체 AI 분석 실행** | 선택된 프로젝트에 대해 성숙도 분석, 기회 발굴, 로드맵 생성, 아키텍처 설계, 거버넌스 분석을 일괄 실행한다. 각 분석은 AI 에이전트가 자동 수행한다. |

AI 컨설팅의 전체 방법론을 개괄적으로 소개하는 페이지이다. 4개의 카드로 구성되어 있다.

### 5.2 AI 성숙도 방법론

**메뉴 경로**: Phase 01 > 컨설팅 방법론 > AI 성숙도 방법론  
**페이지 ID**: `page-maturity-methodology`

📖 **용어 해설**

| 용어 | 설명 |
|------|------|
| **AI 성숙도 모델** | 조직의 AI 활용 수준을 5단계로 평가하는 프레임워크이다. Level 1(초기)부터 Level 5(최적화)까지 구분한다. |
| **진단 (Assessment)** | 현재 조직의 AI 성숙도 수준을 객관적으로 측정하는 과정이다. |

이 페이지는 AI 성숙도를 진단하는 방법론의 상세 내용을 제공한다. 각 레벨별 특성, 진단 항목, 평가 기준이 카드와 테이블로 제시된다.

**AI 성숙도 5단계**:

| 레벨 | 명칭 | 특성 | 실무적 의미 |
|------|------|------|-----------|
| Level 1 | 초기 (Initial) | AI에 대한 인식 부족, 개별 실험 수준 | "AI가 뭔지는 알지만, 우리 업무에 어떻게 쓰는지는 모른다" |
| Level 2 | 탐색 (Exploring) | 파일럿 프로젝트 진행, 데이터 수집 시작 | "한두 개 AI 과제를 시범적으로 해 보고 있다" |
| Level 3 | 정의 (Defined) | AI 전략 수립, 전담 조직 구성, 프로세스 정립 | "AI 전략과 조직이 있고, 여러 부서에서 AI를 도입하고 있다" |
| Level 4 | 관리 (Managed) | AI 성과 측정, MLOps 체계 운영, 전사 확산 | "AI가 핵심 업무에 녹아들어 성과가 측정되고 있다" |
| Level 5 | 최적화 (Optimized) | AI 자율 운영, 지속적 학습, 비즈니스 혁신 | "AI가 스스로 학습하고, 새로운 비즈니스 기회를 창출하고 있다" |

### 5.3 가치-실행 매핑

**메뉴 경로**: Phase 01 > 컨설팅 방법론 > 가치-실행 매핑  
**페이지 ID**: `page-value-mapping`

📖 **용어 해설**

| 용어 | 설명 |
|------|------|
| **가치-실행 용이성 매트릭스** | X축에 "실행 용이성", Y축에 "비즈니스 가치"를 놓고, 발굴된 AI 기회를 4분면에 배치하여 우선순위를 결정하는 도구이다. |
| **Quick Win** | 가치 높음 + 실행 쉬움. 가장 먼저 착수해야 하는 과제이다. |
| **Strategic** | 가치 높음 + 실행 어려움. 중장기 전략으로 추진한다. |
| **Fill-in** | 가치 낮음 + 실행 쉬움. 여유 자원으로 수행한다. |
| **Reconsider** | 가치 낮음 + 실행 어려움. 보류 또는 재검토한다. |

이 페이지에서는 AI 기회 발굴 결과를 2차원 매트릭스에 배치하여 최적의 도입 순서를 결정할 수 있다.

🏭 **실무 시나리오: 제조업 10개 AI 기회 우선순위 결정**

> 생산부에서 10개의 AI 기회가 발굴되었다면:
> - **Quick Win** (먼저 실행): AI 비전 검사 (가치 85점, 구현 쉬움 70점)
> - **Quick Win**: 실시간 생산 모니터링 (가치 75점, 구현 쉬움 65점)
> - **Strategic** (중기 추진): 공정 파라미터 최적화 (가치 95점, 구현 어려움 45점)
> - **Fill-in** (여유 시 실행): 작업지시 자동화 (가치 50점, 구현 쉬움 85점)
> - **Reconsider**: 환경이 갖추어지지 않은 고복잡도 과제

| **리스크 수준** | 발생 가능성 × 영향도로 자동 계산 (1~25) |
| **완화 조치** | 위험을 줄이기 위한 구체적 활동 계획 |
| **잔여 리스크** | 완화 조치 후 남는 위험 수준 |

💡 **DX-to-AX 예시**: 제조업에서 "AI 비전 검사 시스템"을 도입할 때의 리스크 예시:
- **기술적 리스크**: AI 모델의 불량 오탐(false positive)으로 양품 폐기 발생 → 완화: 사람 검사원의 2차 확인 프로세스 유지
- **윤리적 리스크**: AI 판정에 대한 작업자 신뢰도 부족 → 완화: AI 판정 근거(Grad-CAM 시각화) 제공, 교육 프로그램 운영
- **운영적 리스크**: 카메라 고장 시 검사 중단 → 완화: 이중화 카메라 구성, 수동 검사 복귀 프로세스 수립

### 12.2 발굴 결과 해석

"기회 발굴 실행" 버튼을 클릭하면, AI가 입력된 프로세스와 부서 템플릿을 종합 분석하여 AX 기회 목록을 생성한다.

📖 **용어 해설 - 결과 항목**

| 결과 항목 | 단위 | 상세 설명 |
|----------|------|----------|
| **AX 기회명 (title)** | 텍스트 | 발굴된 기회의 명칭이다. 예: "외관 품질 검사 자동화", "수요 예측 AI 고도화" |
| **AX 접근 방법 (ax_approach)** | 텍스트 | AI를 적용하는 구체적 방식이다. "자동화", "AI 품질 개선", "AI 의사결정 지원", "예측 분석" 등 |
| **목표 DX 레벨 (target_dx_level)** | 0~4 | 이 기회를 완료하면 도달하게 되는 DX-to-AX 레벨이다 |
| **ROI 잠재력 점수** | 0~100 | 투자 대비 수익 잠재력 (가중치 30%) |
| **구현 용이성 점수** | 0~100 | 구현의 기술적 용이성 (가중치 25%) |
| **데이터 준비도 점수** | 0~100 | AI 학습 데이터의 가용성 (가중치 25%) |
| **조직 준비도 점수** | 0~100 | 조직의 변화 수용 준비 정도 (가중치 20%) |
| **종합 점수 (overall_score)** | 0~100 | 4개 점수의 가중 합계. 이 점수가 높을수록 우선 추진해야 하는 기회이다 |
| **예상 ROI (estimated_roi)** | % | 투자비 대비 연간 절감의 비율. 150%이면 1억 투자 시 1.5억 절감 |
| **예상 투자비 (estimated_investment)** | 만원 | AI 도입에 필요한 예상 투자 금액 |
| **예상 연간 절감 (estimated_saving)** | 만원 | AI 도입 후 연간 절감 예상 금액 |
| **회수 기간 (payback_months)** | 개월 | 투자비를 절감액으로 회수하는 데 걸리는 기간 |
| **필요 기술 (required_tech)** | 목록 | 구현에 필요한 기술 스택. 예: Python, Computer Vision, YOLO |
| **복잡도 (complexity)** | low/medium/high | 구현 난이도. low=3,000만원, medium=8,000만원, high=2억원 수준 |
| **우선순위 (priority)** | high/medium/low | ROI 잠재력 기반 우선순위 |

**복잡도별 투자비 기준**:

| 복잡도 | 예상 투자비 | 대표 예시 |
|--------|-----------|----------|
| **low** (낮음) | 약 3,000만원 | 발주 자동화, 작업지시 자동화, 통관 자동화 |
| **medium** (보통) | 약 8,000만원 | AI 비전 검사, 수요 예측, 챗봇 구축, SPC 자동화 |
| **high** (높음) | 약 2억원 | 공정 파라미터 최적화, 설계 자동화, 로보어드바이저 |

**AX 기회 자동 발굴 조건**:

시스템은 다음 조건에 해당하는 프로세스에서 자동으로 AX 기회를 식별한다.

| 조건 | 발굴되는 AX 기회 유형 | 목표 DX 레벨 |
|------|---------------------|:-----------:|
| 현재 수행 방법이 "수기 작업" 또는 "엑셀" | **자동화** | Level 2 |
| 오류율이 3% 이상 | **AI 품질 개선** | Level 3 |
| 투입 인력 3명 이상 + 데이터 수집 가능 | **AI 의사결정 지원** | Level 3 |
| 데이터 수집 가능 + 데이터 양이 중간 이상 | **예측 분석** | Level 3 |

🏭 **실무 시나리오: 처음 사용하는 실무자를 위한 단계별 가이드**

> **Step 1**: 사이드바에서 "Phase 04 > AX 기회 발굴"을 클릭한다.  
> **Step 2**: 기업명에 자신의 회사 이름을 입력한다.  
> **Step 3**: 부서를 선택한다. 본인이 속한 부서를 정확히 선택하면, 해당 부서에 맞는 사전 정의 과제가 자동으로 추가된다.  
> **Step 4**: 산업 도메인을 선택한다. 이 선택에 따라 산업별 규제와 Best Practice가 분석에 반영된다.  
> **Step 5**: "분석 대상 프로세스" 영역에서 본인이 담당하는 업무를 하나씩 입력한다.  
> - 프로세스명은 구체적으로: "보고서 작성" (✗) → "월간 품질 보고서 작성" (✓)  
> - 페인포인트는 현실적으로: "수작업" (✗) → "30개 공정 데이터를 수동 복사, 평균 4시간 소요, 오류 빈번" (✓)  
> **Step 6**: "기회 발굴 실행" 버튼을 클릭한다.  
> **Step 7**: 결과를 종합 점수(overall_score) 순으로 확인하고, 가장 높은 점수의 기회부터 검토한다.

#### 13.2.2 생산기술연구소 (4개 과제)

> **제조업 품질관리 팀장의 성장 시나리오**:
> 
> 1. **AX Beginner** (1~3개월): AI 비전 검사 파일럿 등 5건의 입문 과제를 완료한다. "AI 비전 검사가 뭔지는 알겠다."
> 2. **AX Practitioner** (3~6개월): 불량 예측, SPC 자동화 등 20건의 과제를 수행하고, 그중 10건에서 실제 ROI를 달성한다. "스스로 AX 과제를 기획하고 수행할 수 있다."
> 3. **AX Specialist** (6~12개월): 50건의 과제를 완료하고, 품질관리부 전체 업무의 30%를 AX로 전환한다. "우리 부서의 AX를 이끌 수 있다."
> 4. **AX Expert** (1~2년): 100건의 과제, 50건의 ROI 달성, 부서 전환율 50%를 달성한다. "회사 전체의 AX 전략을 제안할 수 있다."
> 5. **AX Master** (2년~): Expert 조건을 충족하면서, 다른 기업의 AX 전환 컨설팅을 수행한 경험이 있다. "제조업 AX의 전문가로 인정받는다."

## 부록 A: 7대 산업 도메인 DX-to-AX 전환 사전

각 산업 도메인별로 핵심 프로세스의 현재 상태(Level 0~1)에서 AX 목표 상태(Level 3~4)까지의 전환 경로를 상세히 기술한다.

### A.1 제조업 전환 사전

### C.2 등급 산정 로직

등급은 **하위 등급부터 순차적으로** 평가하며, 모든 요건을 충족하는 가장 높은 등급이 부여된다.

```
1. Beginner 요건 확인 → 충족 시 Beginner
2. Practitioner 요건 확인 → 충족 시 Practitioner로 승급
3. Specialist 요건 확인 → 충족 시 Specialist로 승급
4. Expert 요건 확인 → 충족 시 Expert로 승급
5. Master 요건 확인 → 충족 시 (외부 컨설팅 경험 필수) Master로 승급
```
