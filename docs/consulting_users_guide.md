## 1. 개요

본 문서는 100K-AX Expert Platform을 활용한 기업 AI 전환(AX: AI Transformation) 컨설팅 수행 시 필요한 입력 항목과 각 항목이 컨설팅 단계에 미치는 영향에 대하여 기술한다. 플랫폼은 5단계 컨설팅 프레임워크와 국제 표준(ISO 42001, ISO 38500, ISO 24030, ISO 23053)을 기반으로 체계적인 AI 도입 컨설팅을 지원한다.

### 1.1 플랫폼 구성 요소

본 플랫폼은 다음과 같은 핵심 구성 요소로 이루어져 있다.

| 구성 요소 | 설명 |
|-----------|------|
| 5단계 프레임워크 | AI 비전 수립부터 운영까지의 전체 컨설팅 생명주기 관리 |
| ISO 표준 체크리스트 | 국제 표준 기반 거버넌스 및 역량 평가 도구 |
| AI 에이전트 시스템 | 멀티 에이전트 기반 자동 분석 및 보고서 생성 |
| 시나리오 분석 | 보수적/균형/적극적 시나리오 자동 생성 및 비교 |

## 2. 플랫폼 메뉴 구조

플랫폼은 단계별 컨설팅 진행을 위하여 다음과 같은 메뉴 구조로 구성되어 있다.

### 2.1 시작 메뉴 (PHASE START)

| 메뉴명 | 기능 설명 |
|--------|-----------|
| 대시보드 | 프로젝트 현황, AI 성숙도 진단 결과 요약, 워크플로우 진행 상태 표시 |
| 새 프로젝트 | 신규 컨설팅 프로젝트 생성 및 기업 프로필 입력 |

| 메뉴명 | 기능 설명 |
|--------|-----------|
| 방법론 개요 | 세부 컨설팅 방법론에 대한 개요 확인 |
| 성숙도 진단 방법 | 성숙도 진단 세부 방법론 및 평가 기준 안내 |
| 가치-실행 매핑 | AI 과제의 비즈니스 가치와 실행 용이성 평가 매트릭스 |

| 단계 | 메뉴명 | 기능 설명 |
|------|--------|-----------|

| Stage 2 | 요건 & 아키텍처 설계 | 상세 요건 정의, 기술 아키텍처 설계, 거버넌스 체계 수립 |

| ISO 24030 | 평가 대시보드, 성숙도 진단 설문, AI 시스템 인벤토리, 위험 평가 매트릭스, 공정성 메트릭, 거버넌스 체크리스트, 개선 로드맵 |

## 3. 컨설팅 진행 절차

컨설팅은 다음 5단계 프레임워크에 따라 순차적으로 진행된다. 각 단계에서 입력된 데이터는 다음 단계의 분석 및 의사결정에 직접적인 영향을 미친다.

### 3.1 Stage 1: AI 비전 및 전략 수립

**목적**: 고객사의 AI 도입 목표와 전사적 방향을 정의하고, AI가 비즈니스에 제공할 수 있는 최대 가치를 도출한다.

#### 3.1.1 AI 성숙도 진단 (Maturity Assessment)

성숙도 진단은 4대 핵심 영역에 대하여 1~5점 척도로 평가한다.

**전략 및 비전 영역 입력 항목**

| 입력 항목 | 설명 | 평가 기준 |
|-----------|------|-----------|
| AI 비전 및 목표의 명확성 | 조직의 AI 비전이 얼마나 명확하게 정의되어 있는가 | 1:초기 ~ 5:최적화 |
| AI 투자 및 예산 관리 체계 | AI 관련 투자 및 예산 관리가 체계적으로 이루어지는가 | 1:초기 ~ 5:최적화 |
| AI 활용 사례 포트폴리오 관리 | AI 활용 사례들이 포트폴리오로 관리되고 있는가 | 1:초기 ~ 5:최적화 |
| AI ROI 측정 체계 | AI 투자 대비 성과를 측정하는 체계가 있는가 | 1:초기 ~ 5:최적화 |

**조직 및 역량 영역 입력 항목**

| 입력 항목 | 설명 | 평가 기준 |
|-----------|------|-----------|
| AI 전담 조직 및 역할 정의 | AI 전담 조직이 존재하고 역할이 정의되어 있는가 | 1:초기 ~ 5:최적화 |
| AI 인력 확보 및 역량 수준 | 필요한 AI 인력이 확보되어 있고 역량 수준이 적절한가 | 1:초기 ~ 5:최적화 |
| AI 교육 및 역량 개발 프로그램 | 체계적인 AI 교육 프로그램이 운영되고 있는가 | 1:초기 ~ 5:최적화 |
| 변화 관리 및 AI 문화 | AI 도입에 따른 변화 관리와 문화가 조성되어 있는가 | 1:초기 ~ 5:최적화 |

**데이터 및 기술 영역 입력 항목**

| 입력 항목 | 설명 | 평가 기준 |
|-----------|------|-----------|
| 데이터 인프라 및 접근성 | 데이터 인프라가 구축되어 있고 접근이 용이한가 | 1:초기 ~ 5:최적화 |
| 데이터 품질 및 거버넌스 | 데이터 품질 관리와 거버넌스가 수립되어 있는가 | 1:초기 ~ 5:최적화 |
| MLOps 플랫폼 활용 수준 | MLOps 플랫폼이 도입되어 활용되고 있는가 | 1:초기 ~ 5:최적화 |
| 클라우드 활용 및 확장성 | 클라우드 인프라가 활용되고 확장 가능한가 | 1:초기 ~ 5:최적화 |

**프로세스 및 거버넌스 영역 입력 항목**

| 입력 항목 | 설명 | 평가 기준 |
|-----------|------|-----------|
| AI 개발 방법론 표준화 | AI 개발 방법론이 표준화되어 있는가 | 1:초기 ~ 5:최적화 |
| 모델 검증 및 배포 승인 절차 | 모델 검증과 배포 승인 절차가 수립되어 있는가 | 1:초기 ~ 5:최적화 |
| AI 윤리 및 리스크 관리 | AI 윤리 원칙과 리스크 관리 체계가 있는가 | 1:초기 ~ 5:최적화 |
| 모니터링 및 운영 체계 | AI 모델 모니터링과 운영 체계가 구축되어 있는가 | 1:초기 ~ 5:최적화 |

**입력 데이터의 영향**:
- 성숙도 진단 결과는 레이더 차트로 시각화되어 현재 수준과 목표 수준의 Gap 분석에 활용된다.
- 각 영역별 점수는 AI 에이전트의 개선 권고사항 도출에 직접적으로 활용된다.
- 성숙도 수준에 따라 로드맵 단계별 우선순위가 결정된다.

#### 3.1.2 AI 기회 발굴 (Opportunity Identification)

비즈니스 가치 사슬 분석을 통해 AI 적용 가능 영역을 식별한다.

**기회 발굴 입력 항목**

| 입력 항목 | 설명 | 선택 옵션 |
|-----------|------|-----------|
| 비즈니스 영역 | AI 적용 대상 비즈니스 영역 | 생산/제조, 영업/마케팅, 고객 서비스, 물류/SCM, 인사/총무, 재무/회계, 연구개발, 품질관리 |
| 해결하고자 하는 문제점 | 현재 프로세스의 Pain Point | 자유 기술 |
| 현재 프로세스 설명 | 현재 업무 프로세스 상세 설명 | 자유 기술 |
| 기대 효과 | AI 도입 시 기대되는 효과 | 자유 기술 |
| 데이터 가용성 | 필요 데이터의 확보 수준 | 1~5점 |
| 긴급도 | 문제 해결의 긴급성 | 1~5점 |
| 전략 정합성 | 기업 전략과의 부합도 | 1~5점 |

**입력 데이터의 영향**:
- 데이터 가용성, 긴급도, 전략 정합성 점수는 가치-실행 용이성 매트릭스 분류(Quick Win, Strategic, Fill-in, Reconsider)에 활용된다.
- 식별된 기회 목록은 Stage 2의 Use Case 선정 기준으로 활용된다.
- AI 에이전트는 입력된 정보를 기반으로 추가 기회 영역을 제안한다.

#### 3.1.3 전략 및 로드맵 (Strategy & Roadmap)

**로드맵 입력 항목**

| 입력 항목 | 설명 |
|-----------|------|
| AI 비전 선언문 | 조직의 AI 비전을 명확하게 선언하는 문장 |
| 전략적 목표 | 달성하고자 하는 전략적 목표 목록 |
| 핵심 성과 지표(KPI) | KPI명, 현재값, 목표값 |
| Quick Win 과제 | 6개월 이내 완료 가능한 높은 ROI 과제 |
| Strategic 과제 | 6-18개월 소요되는 핵심 역량 구축 과제 |
| Transformational 과제 | 18개월 이상 소요되는 비즈니스 모델 혁신 과제 |

**입력 데이터의 영향**:
- 비전 선언문과 전략적 목표는 전체 컨설팅의 방향성을 설정한다.
- KPI는 Stage 5 모니터링 단계의 성과 측정 기준으로 활용된다.
- 로드맵 과제 분류는 시나리오(보수적/균형/적극적) 생성의 기초 데이터가 된다.

### 3.2 Stage 2: Use Case 및 설계 정의

**목적**: 우선순위가 높은 과제에 대하여 구체적인 AI 시스템/솔루션 설계를 진행한다.

#### 3.2.1 상세 요건 정의 (Requirements Definition)

**요건 정의 입력 항목**

| 입력 항목 | 설명 |
|-----------|------|
| Use Case 선택 | Stage 1에서 발굴한 기회 중 선택 |
| 기능 요건 | AI 시스템이 수행해야 하는 기능 명세 |
| 비기능 요건 | 성능, 보안, 가용성 등 품질 요건 |
| 데이터 요건 | 필요 데이터 종류, 형식, 수집 방법 |
| 성공 기준 | Use Case 성공 판단 기준 지표 |

**입력 데이터의 영향**:
- 상세 요건은 Stage 3 PoC 범위 설정의 기준이 된다.
- 비기능 요건은 아키텍처 설계 시 기술 선택에 직접적인 영향을 미친다.
- 성공 기준은 파일럿 평가 및 확산 의사결정의 판단 기준으로 활용된다.

#### 3.2.2 기술 및 아키텍처 설계 (Technical Architecture)

**아키텍처 설계 입력 항목**

| 구분 | 입력 항목 | 설명 |
|------|-----------|------|
| 데이터 아키텍처 | 데이터 소스 | 데이터 원천 시스템 목록 |
| | ETL 방식 | 데이터 추출/변환/적재 방식 |
| | 저장소 유형 | 데이터 저장소 종류 (Data Lake, DW 등) |
| | 처리 방법 | 배치/실시간/스트리밍 |
| ML 아키텍처 | 모델 유형 | 적용할 ML/AI 모델 종류 |
| | 학습 방식 | 지도학습/비지도학습/강화학습 등 |
| | 배포 방식 | 온프레미스/클라우드/하이브리드 |
| | 추론 엔드포인트 | API 서빙 방식 |

## 4. ISO 표준 기반 거버넌스 평가

플랫폼은 4개의 국제 표준을 기반으로 AI 거버넌스를 평가한다.

### 4.1 ISO 42001 AI 관리 시스템

ISO/IEC 42001은 AI 관리 시스템(AIMS: AI Management System)에 대한 국제 표준이다.

#### 4.1.1 거버넌스 3대 핵심 영역 평가

**영역별 입력 항목**

| 영역 | 입력 항목 | 선택 옵션 |
|------|-----------|-----------|
| 전략 및 정책 | AI 윤리 원칙 수립 현황 | 미수립, 초안 단계, 승인 완료, 전사 시행 중 |
| | 데이터 정책 수립 현황 | 미수립, 초안 단계, 승인 완료, 전사 시행 중 |
| 프로세스 및 통제 | AI 개발 프로세스 표준화 수준 | 표준화 안됨, 부분 표준화, 전체 표준화, 최적화됨 |

## 5. 분석 및 산출물

### 5.1 시나리오 분석

플랫폼은 입력된 데이터를 기반으로 3가지 시나리오를 자동 생성한다.

| 시나리오 | 특성 | 활용 상황 |
|----------|------|-----------|
| 보수적 (Conservative) | 낮은 투자, 낮은 리스크, Quick Win 중심 | 예산 제약, 조직 저항이 큰 경우 |
| 균형 (Balanced) | 적정 투자, 적정 리스크, Quick Win + Strategic | 일반적인 권장 시나리오 |
| 적극적 (Aggressive) | 높은 투자, 높은 리스크, 전체 로드맵 | 경쟁 압박, 빠른 전환 필요 시 |

**시나리오 생성에 영향을 미치는 입력 데이터**:
- Stage 1 성숙도 진단 결과
- 발굴된 기회 목록 및 우선순위
- 로드맵 과제 분류 (Quick Win/Strategic/Transformational)
- 가용 예산 및 인력
- 리스크 허용 수준

## 6. 입력 데이터의 컨설팅 단계별 영향 분석

### 6.1 데이터 흐름 및 연계 관계

다음은 각 단계에서 입력된 데이터가 후속 단계에 미치는 영향을 정리한 것이다.

```
Stage 1 (전략 수립)
├── 성숙도 진단 결과 → Stage 2 요건 정의 시 현실적 목표 설정
├── 기회 발굴 목록 → Stage 2 Use Case 선정 기준
├── 로드맵 과제 분류 → 시나리오 생성 기초 데이터
└── KPI 정의 → Stage 5 모니터링 성과 측정 기준

Stage 2 (설계 정의)
├── 상세 요건 → Stage 3 PoC 범위 설정
├── 아키텍처 설계 → Stage 3 플랫폼 구축 청사진
├── 거버넌스 체계 → ISO 42001 체크리스트 평가
└── 성공 기준 → Stage 4 파일럿 평가 기준

Stage 3 (솔루션 구축)
├── PoC 결과 → Stage 4 파일럿 진행 여부 결정
├── 플랫폼 구성 → ISO 23053 체크리스트 평가
├── 통합 설계 → Stage 4 확산 시 안정성
└── 투자 비용 → ROI 분석 및 시나리오 비교

Stage 4 (파일럿 및 확산)
├── 파일럿 결과 → 확산 의사결정 근거
├── 변화 관리 → 사용자 수용도 및 활용률
├── 확산 계획 → Stage 5 운영 범위
└── 성과 데이터 → 최종 보고서 핵심 지표

Stage 5 (운영 및 개선)
├── 모니터링 데이터 → 지속적 개선 피드백
├── 재학습 트리거 → 모델 성능 유지
├── 거버넌스 검토 → ISO 42001 인증 준비
└── 감사 결과 → 규제 준수 증빙
```

### 6.2 핵심 입력 항목별 영향도 분석

다음은 각 입력 항목이 전체 컨설팅에 미치는 영향도를 분석한 것이다.

| 입력 항목 | 영향받는 단계 | 영향도 | 설명 |
|-----------|---------------|--------|------|
| AI 성숙도 점수 | 전 단계 | 높음 | 현재 수준 파악 및 목표 설정의 기준 |
| 기회 목록 및 우선순위 | Stage 2, 3, 4 | 높음 | Use Case 선정 및 실행 순서 결정 |

| PoC 성공 기준 | Stage 3, 4 | 높음 | 파일럿 진행 여부 결정의 핵심 |
| 거버넌스 체계 | ISO 평가, Stage 5 | 높음 | 인증 준비 및 지속적 운영의 기반 |
| 변화 관리 계획 | Stage 4 | 중간 | 확산 성공률에 직접 영향 |
| 모니터링 지표 | Stage 5 | 중간 | 운영 효율성 및 개선 방향 결정 |
| KPI 정의 | Stage 1, 5 | 중간 | 성과 측정 및 ROI 계산 기준 |
| 인력/리소스 계획 | Stage 3, 4 | 중간 | TCO 및 실행 가능성에 영향 |
| 규제 준수 항목 | ISO 평가, Stage 5 | 중간 | 법적 리스크 관리 |

### 6.3 입력 데이터 품질이 결과에 미치는 영향

입력 데이터의 품질은 AI 에이전트 분석 결과의 정확도와 컨설팅 권고사항의 적절성에 직접적인 영향을 미친다.

**데이터 품질 요소별 영향**

| 구체성 | 일반적/추상적 권고 도출 | 구체적 수치 및 사례 기반 입력 |
