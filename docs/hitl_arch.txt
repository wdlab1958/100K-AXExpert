2. HITL 아키텍처 개요
================================================================================

[설계 철학]
"AI는 제안하고, 인간은 결정한다"
- 모든 중요한 의사결정은 인간이 승인
- 에이전트는 자율적으로 분석하되, 제어 포인트에서 대기
- 전 과정이 투명하고 감사 가능

[HITL 레이어 구조]
┌────────────────────────────────────┐
│  Layer 1: Human Control Interface (Frontend)                │
│  - Dashboard, Approval UI, Monitoring Console               │
├────────────────────────────────────┤
│  Layer 2: HITL Orchestration Layer (New)                    │
│  - Checkpoint Manager                                       │
│  - Approval Workflow Engine                                 │
│  - Human Task Queue                                         │
│  - Escalation Manager                                       │
├────────────────────────────────────┤
│  Layer 3: Agent Execution Layer (Existing + Enhanced)       │
│  - Consulting Agents with Pause/Resume                      │
│  - State Persistence                                        │
│  - Rollback Capability                                      │
├────────────────────────────────────┤
│  Layer 4: Audit & Compliance Layer (Enhanced)               │
│  - Comprehensive Audit Trail                                │
│  - Decision Tracking                                        │
│  - Explainability Engine                                    │
└────────────────────────────────────┘

[주요 HITL 패턴]
1. Approval Gates: 중요 단계 진입 전 인간 승인 필요
2. Human Override: 에이전트 결과를 인간이 수정 가능
3. Active Monitoring: 실시간 에이전트 활동 모니터링 및 개입
4. Fallback Escalation: 에이전트 불확실성 시 인간 에스컬레이션

3. 핵심 컴포넌트 설계
================================================================================

[3.1 Checkpoint Manager]
목적: 워크플로우의 중요 지점에서 인간 개입 제어

├── CheckpointType (Enum)
│   ├── PRE_STAGE: 단계 시작 전 승인
│   ├── POST_STAGE: 단계 완료 후 검증
│   ├── CRITICAL_DECISION: 중요 의사결정 시점
│   ├── HIGH_RISK: 고위험 작업 전
│   └── QUALITY_CHECK: 품질 검증 필요

├── CheckpointStatus (Enum)
│   ├── PENDING: 인간 승인 대기
│   ├── APPROVED: 승인됨, 진행 가능
│   ├── REJECTED: 거부됨, 재작업 필요
│   ├── MODIFIED: 수정 후 승인
│   └── SKIPPED: 건너뜀 (긴급 상황)

└── Checkpoint Model
    ├── id: str
    ├── project_id: str
    ├── checkpoint_type: CheckpointType
    ├── stage: ConsultingStage
    ├── agent_id: str
    ├── context: Dict[str, Any]  # 의사결정 컨텍스트
    ├── agent_output: Optional[Dict]  # 에이전트 출력
    ├── status: CheckpointStatus
    ├── reviewer_id: Optional[str]
    ├── reviewer_comment: Optional[str]
    ├── created_at: datetime
    ├── reviewed_at: Optional[datetime]
    └── timeout_seconds: int  # 자동 에스컬레이션


[3.2 Human Task Queue]
목적: 인간 검토가 필요한 작업을 우선순위별로 관리

├── TaskPriority (Enum)
│   ├── CRITICAL: 즉시 처리 필요 (15분 이내)
│   ├── HIGH: 긴급 (1시간 이내)
│   ├── MEDIUM: 일반 (4시간 이내)
│   └── LOW: 낮음 (1일 이내)

├── TaskType (Enum)
│   ├── APPROVAL: 승인 요청
│   ├── REVIEW: 검토 요청
│   ├── VALIDATION: 검증 요청
│   ├── CORRECTION: 수정 요청
│   └── ESCALATION: 에스컬레이션

└── HumanTask Model
    ├── id: str
    ├── project_id: str
    ├── task_type: TaskType
    ├── priority: TaskPriority
    ├── title: str
    ├── description: str
    ├── context: Dict[str, Any]
    ├── assignee_role: str  # "consultant", "manager", "executive"
    ├── assignee_id: Optional[str]
    ├── status: str  # "pending", "in_progress", "completed", "cancelled"
    ├── deadline: datetime
    ├── created_at: datetime
    └── completed_at: Optional[datetime]


[3.3 Approval Workflow Engine]
목적: 복잡한 다단계 승인 프로세스 자동화

Workflow Types:
1. Single Approval: 단일 승인자 (컨설턴트)
2. Hierarchical Approval: 계층적 승인 (컨설턴트 → 매니저 → 임원)
3. Parallel Approval: 병렬 승인 (여러 부서장 동시)
4. Conditional Approval: 조건부 승인 (금액/리스크 기준)

├── ApprovalWorkflow Model
│   ├── id: str
│   ├── name: str
│   ├── description: str
│   ├── workflow_type: str
│   ├── steps: List[ApprovalStep]
│   ├── conditions: Dict[str, Any]
│   └── timeout_policy: str

└── ApprovalStep Model
    ├── step_id: str
    ├── step_name: str
    ├── required_role: str
    ├── required_approvers: int  # 최소 승인자 수
    ├── auto_approve_conditions: Optional[Dict]
    └── timeout_action: str  # "escalate", "auto_approve", "reject"


[3.4 Agent State Manager]
목적: 에이전트 실행 상태를 체크포인트별로 저장 및 복원

Features:
- Pause/Resume: 에이전트 실행 일시정지 및 재개
- Rollback: 이전 체크포인트로 되돌리기
- State Snapshot: 각 단계별 상태 스냅샷 저장
- Branch & Merge: 여러 시나리오 병렬 탐색

├── AgentExecutionState Model
│   ├── id: str
│   ├── project_id: str
│   ├── agent_id: str
│   ├── execution_id: str  # 실행 세션 ID
│   ├── status: str  # "running", "paused", "waiting_approval", "completed", "failed"
│   ├── current_step: str
│   ├── state_snapshot: Dict[str, Any]
│   ├── variables: Dict[str, Any]
│   ├── messages: List[AgentMessage]
│   ├── checkpoints: List[str]  # checkpoint IDs
│   ├── can_rollback: bool
│   ├── created_at: datetime
│   └── updated_at: datetime


[3.5 Escalation Manager]
목적: 에이전트가 처리 불가능한 상황 시 인간에게 에스컬레이션

Escalation Triggers:
- Confidence Score < Threshold (예: 0.7)
- Contradictory Information Detected
- High Risk Decision Required
- Timeout or Deadlock
- Explicit Agent Request

├── EscalationRule Model
│   ├── id: str
│   ├── name: str
│   ├── trigger_condition: str  # Python expression
│   ├── escalate_to_role: str
│   ├── priority: TaskPriority
│   ├── auto_notify: bool
│   └── notification_channels: List[str]  # ["email", "slack", "ui"]

└── EscalationCase Model
    ├── id: str
    ├── project_id: str
    ├── agent_id: str
    ├── trigger_reason: str
    ├── context: Dict[str, Any]
    ├── suggested_actions: List[str]
    ├── escalated_to: str
    ├── status: str
    ├── resolution: Optional[str]
    ├── created_at: datetime
    └── resolved_at: Optional[datetime]


[3.6 Audit Trail Engine]
목적: 모든 의사결정과 인간 개입을 상세히 기록

Audit Event Types:
- AGENT_STARTED, AGENT_COMPLETED, AGENT_FAILED
- CHECKPOINT_CREATED, CHECKPOINT_APPROVED, CHECKPOINT_REJECTED
- HUMAN_OVERRIDE, HUMAN_MODIFICATION
- STATE_ROLLBACK, WORKFLOW_RESUMED
- ESCALATION_TRIGGERED, ESCALATION_RESOLVED

├── AuditEvent Model (Enhanced)
│   ├── event_id: str
│   ├── timestamp: datetime
│   ├── event_type: str
│   ├── actor_type: str  # "human", "agent", "system"
│   ├── actor_id: str
│   ├── project_id: str
│   ├── resource_type: str
│   ├── resource_id: str
│   ├── action: str
│   ├── before_state: Optional[Dict]
│   ├── after_state: Optional[Dict]
│   ├── reason: Optional[str]
│   ├── confidence_score: Optional[float]  # AI 신뢰도
│   ├── decision_factors: Optional[List[str]]
│   └── trace_id: str  # 전체 워크플로우 추적


[3.7 Explainability Engine]
목적: AI 에이전트의 결정에 대한 설명 생성

Features:
- Decision Explanation: 왜 이 결정을 했는지
- Factor Analysis: 어떤 요인이 가장 중요했는지
- Alternative Scenarios: 다른 선택지는 무엇이었는지
- Confidence Breakdown: 신뢰도 구성 요소

└── Explanation Model
    ├── decision_id: str
    ├── agent_id: str
    ├── decision_summary: str
    ├── reasoning_steps: List[str]
    ├── key_factors: List[Dict]  # [{factor, weight, impact}]
    ├── alternatives_considered: List[Dict]
    ├── confidence_score: float
    ├── confidence_breakdown: Dict[str, float]
    ├── data_sources: List[str]
    ├── assumptions: List[str]
    ├── limitations: List[str]
    └── generated_at: datetime

5. 승인 워크플로우 (Approval Workflows)
================================================================================

[워크플로우 템플릿]

Template 1: 단일 승인 (Simple Approval)
─────────────────────────────────────────
Use Case: 일반적인 분석 결과 검토
Flow:
  Start → Consultant Review → [Approved/Rejected] → End
Timeout: 4시간
Auto-escalate: Manager

Template 2: 2단계 승인 (Two-Level Approval)
─────────────────────────────────────────
Use Case: 중요 설계 결정
Flow:
  Start → Technical Lead Review → Manager Approval → End
Timeout: 1일
Auto-escalate: Director

Template 3: 임원 승인 (Executive Approval)
─────────────────────────────────────────
Use Case: 전략적 의사결정, 대규모 투자
Flow:
  Start → Consultant → Manager → Director → Executive → End
Parallel: 재무, 법무, IT 부서 동시 검토 (Stage 2)
Timeout: 3일
Auto-escalate: CEO

Template 4: 조건부 승인 (Conditional Approval)
─────────────────────────────────────────
Use Case: 예산 기반 승인 권한
Conditions:
  - 예산 < 1억: Manager 승인
  - 예산 1억~5억: Director 승인
  - 예산 > 5억: Executive 승인
Flow: Dynamic based on conditions

Template 5: 병렬 승인 (Parallel Approval)
─────────────────────────────────────────
Use Case: 타 부서 동의 필요
Flow:
  Start → [IT Lead, Business Lead, Security Lead] (병렬) → Manager → End
Required: 모든 부서 승인 필요 (AND logic)
Timeout: 2일


[승인 결정 옵션]
1. Approve: 승인, 다음 단계 진행
2. Approve with Modifications: 수정 사항 반영 후 승인
3. Reject: 거부, 에이전트 재실행
4. Request More Info: 추가 정보 요청
5. Escalate: 상위 의사결정권자에게 에스컬레이션
6. Defer: 보류, 나중에 검토


================================================================================
6. 감사 시스템 (Audit System)
================================================================================

[감사 수준 (Audit Levels)]

Level 1: Basic Audit
- 누가, 언제, 무엇을 했는지만 기록
- Event: action, timestamp, actor

Level 2: Standard Audit (현재 구현)
- 상태 변경 전후 기록
- Event: action, timestamp, actor, before, after

Level 3: Comprehensive Audit (HITL 필수)
- 의사결정 근거, 대안, 신뢰도까지 기록
- Event: action, timestamp, actor, before, after, reason, alternatives, confidence

Level 4: Forensic Audit
- 완전한 재현 가능, 모든 컨텍스트 저장
- Event: full context snapshot, all variables, messages, external calls


[감사 보고서 종류]

1. Decision Trail Report
   - 특정 의사결정의 전체 과정 추적
   - 누가 언제 어떤 근거로 결정했는지

2. Human Intervention Report
   - 인간 개입 빈도, 유형, 결과 분석
   - 에이전트 신뢰도 개선 인사이트

3. Compliance Report
   - 규정 준수 여부 검증
   - 필수 승인 절차 이행 확인

4. Agent Performance Report
   - 에이전트별 성공률, 인간 수정 빈도
   - 개선이 필요한 에이전트 식별

5. Timeline Report
   - 프로젝트 전체 타임라인
   - 병목 지점 및 지연 원인 분석


[감사 데이터 보존]
- Real-time: In-memory cache (최근 1시간)
- Short-term: SQLite DB (최근 30일)
- Long-term: 파일 아카이브 (1년)
- Compliance: 암호화된 불변 저장소 (법적 요구사항에 따라)

# ==================== HITL Core Models ====================

class CheckpointType(str, Enum):
    PRE_STAGE = "pre_stage"
    POST_STAGE = "post_stage"
    CRITICAL_DECISION = "critical_decision"
    HIGH_RISK = "high_risk"
    QUALITY_CHECK = "quality_check"

class CheckpointStatus(str, Enum):
    PENDING = "pending"
    APPROVED = "approved"
    REJECTED = "rejected"
    MODIFIED = "modified"
    SKIPPED = "skipped"
    TIMEOUT = "timeout"

class Checkpoint(BaseModel):
    id: str = Field(default_factory=lambda: str(uuid.uuid4()))
    project_id: str
    checkpoint_type: CheckpointType
    stage: ConsultingStage
    agent_id: str
    agent_name: str

    # Context
    title: str
    description: str
    context: Dict[str, Any] = Field(default_factory=dict)
    agent_output: Optional[Dict[str, Any]] = None
    agent_confidence: Optional[float] = None  # 0.0 ~ 1.0

    # Status
    status: CheckpointStatus = CheckpointStatus.PENDING

    # Review
    required_role: str = "consultant"  # consultant, manager, director, executive
    reviewer_id: Optional[str] = None
    reviewer_name: Optional[str] = None
    reviewer_comment: Optional[str] = None
    review_decision: Optional[str] = None  # approve, reject, modify, escalate
    modified_output: Optional[Dict[str, Any]] = None

    # Timing
    created_at: datetime = Field(default_factory=datetime.now)
    reviewed_at: Optional[datetime] = None
    timeout_seconds: int = 14400  # 4 hours default
    deadline: datetime = Field(default_factory=lambda: datetime.now() + timedelta(hours=4))

    # Workflow
    blocking: bool = True  # True = must wait for approval, False = async review
    auto_approve_on_timeout: bool = False
    escalate_on_timeout: bool = True
    escalate_to_role: Optional[str] = "manager"

class TaskPriority(str, Enum):
    CRITICAL = "critical"  # 15 min
    HIGH = "high"  # 1 hour
    MEDIUM = "medium"  # 4 hours
    LOW = "low"  # 1 day

class TaskType(str, Enum):
    APPROVAL = "approval"
    REVIEW = "review"
    VALIDATION = "validation"
    CORRECTION = "correction"
    ESCALATION = "escalation"

class HumanTask(BaseModel):
    id: str = Field(default_factory=lambda: str(uuid.uuid4()))
    project_id: str
    checkpoint_id: Optional[str] = None  # Link to checkpoint if applicable

    # Task Info
    task_type: TaskType
    priority: TaskPriority
    title: str
    description: str
    context: Dict[str, Any] = Field(default_factory=dict)

    # Assignment
    assignee_role: str  # consultant, manager, director, executive
    assignee_id: Optional[str] = None
    assignee_name: Optional[str] = None

    # Status
    status: str = "pending"  # pending, assigned, in_progress, completed, cancelled

    # Actions
    available_actions: List[str] = Field(default_factory=list)  # ["approve", "reject", "modify"]
    response_data: Optional[Dict[str, Any]] = None

    # Timing
    created_at: datetime = Field(default_factory=datetime.now)
    assigned_at: Optional[datetime] = None
    deadline: datetime = Field(default_factory=lambda: datetime.now() + timedelta(hours=4))
    completed_at: Optional[datetime] = None

    # Escalation
    escalated_from: Optional[str] = None  # Previous task ID if escalated
    can_escalate: bool = True
    escalate_to_role: Optional[str] = None

class AgentExecutionState(BaseModel):
    id: str = Field(default_factory=lambda: str(uuid.uuid4()))
    project_id: str
    agent_id: str
    execution_id: str = Field(default_factory=lambda: str(uuid.uuid4()))

    # Status
    status: str = "running"  # running, paused, waiting_approval, completed, failed, cancelled
    current_step: str
    progress: float = 0.0  # 0.0 ~ 1.0

    # State
    state_snapshot: Dict[str, Any] = Field(default_factory=dict)
    variables: Dict[str, Any] = Field(default_factory=dict)
    messages: List[Dict[str, Any]] = Field(default_factory=list)

    # Control
    can_pause: bool = True
    can_resume: bool = False
    can_rollback: bool = False
    can_cancel: bool = True

    # Checkpoints
    checkpoint_ids: List[str] = Field(default_factory=list)
    current_checkpoint_id: Optional[str] = None

    # Timing
    created_at: datetime = Field(default_factory=datetime.now)
    updated_at: datetime = Field(default_factory=datetime.now)
    completed_at: Optional[datetime] = None

    # Error handling
    error: Optional[str] = None
    retry_count: int = 0
    max_retries: int = 3

class EscalationTrigger(str, Enum):
    LOW_CONFIDENCE = "low_confidence"
    TIMEOUT = "timeout"
    HIGH_RISK = "high_risk"
    AGENT_REQUEST = "agent_request"
    QUALITY_ISSUE = "quality_issue"
    CONTRADICTION = "contradiction"
    EXTERNAL_REQUIREMENT = "external_requirement"

class EscalationCase(BaseModel):
    id: str = Field(default_factory=lambda: str(uuid.uuid4()))
    project_id: str
    agent_id: str
    checkpoint_id: Optional[str] = None

    # Trigger
    trigger: EscalationTrigger
    trigger_reason: str
    trigger_details: Dict[str, Any] = Field(default_factory=dict)

    # Context
    context: Dict[str, Any] = Field(default_factory=dict)
    agent_output: Optional[Dict[str, Any]] = None
    confidence_score: Optional[float] = None

    # Escalation
    escalated_from_role: str
    escalated_to_role: str
    escalated_to_id: Optional[str] = None

    # Resolution
    status: str = "open"  # open, assigned, in_progress, resolved, cancelled
    suggested_actions: List[str] = Field(default_factory=list)
    resolution: Optional[str] = None
    resolution_details: Optional[Dict[str, Any]] = None

    # Timing
    created_at: datetime = Field(default_factory=datetime.now)
    assigned_at: Optional[datetime] = None
    resolved_at: Optional[datetime] = None

class DecisionExplanation(BaseModel):
    decision_id: str
    checkpoint_id: Optional[str] = None
    agent_id: str

    # Explanation
    decision_summary: str
    reasoning_steps: List[str]
    key_factors: List[Dict[str, Any]]  # [{name, weight, impact, value}]

    # Alternatives
    alternatives_considered: List[Dict[str, Any]]
    why_chosen: str
    why_not_alternatives: List[str]

    # Confidence
    confidence_score: float  # 0.0 ~ 1.0
    confidence_breakdown: Dict[str, float]  # {factor: contribution}

    # Sources
    data_sources: List[str]
    assumptions: List[str]
    limitations: List[str]

    # Meta
    generated_at: datetime = Field(default_factory=datetime.now)
    llm_model: Optional[str] = None

[HITL API Endpoints]

# ==================== Checkpoint Management ====================

POST   /api/v1/hitl/checkpoints
       - Create new checkpoint
       Request: {project_id, checkpoint_type, stage, agent_id, context, ...}
       Response: {checkpoint_id, status, deadline, ...}

GET    /api/v1/hitl/checkpoints/{checkpoint_id}
       - Get checkpoint details

PUT    /api/v1/hitl/checkpoints/{checkpoint_id}/review
       - Review checkpoint (approve/reject/modify)
       Request: {decision, comment, modified_output, ...}
       Response: {status, next_action, ...}

GET    /api/v1/hitl/checkpoints
       - List checkpoints (with filters)
       Query: ?project_id=&status=&assignee_role=

DELETE /api/v1/hitl/checkpoints/{checkpoint_id}
       - Cancel/skip checkpoint


# ==================== Human Task Queue ====================

GET    /api/v1/hitl/tasks
       - Get human tasks (filtered by role, status, priority)
       Query: ?assignee_role=consultant&status=pending&priority=high

GET    /api/v1/hitl/tasks/{task_id}
       - Get task details

POST   /api/v1/hitl/tasks/{task_id}/assign
       - Assign task to user
       Request: {assignee_id, assignee_name}

POST   /api/v1/hitl/tasks/{task_id}/complete
       - Complete task with response
       Request: {response_data, comment}

POST   /api/v1/hitl/tasks/{task_id}/escalate
       - Escalate task to higher authority
       Request: {escalate_to_role, reason}


# ==================== Agent Execution Control ====================

GET    /api/v1/hitl/agents/executions/{execution_id}
       - Get agent execution state

POST   /api/v1/hitl/agents/executions/{execution_id}/pause
       - Pause agent execution

POST   /api/v1/hitl/agents/executions/{execution_id}/resume
       - Resume agent execution

POST   /api/v1/hitl/agents/executions/{execution_id}/cancel
       - Cancel agent execution

POST   /api/v1/hitl/agents/executions/{execution_id}/rollback
       - Rollback to previous checkpoint
       Request: {checkpoint_id}

GET    /api/v1/hitl/agents/executions
       - List agent executions
       Query: ?project_id=&agent_id=&status=


# ==================== Approval Workflows ====================

GET    /api/v1/hitl/workflows
       - List approval workflow templates

GET    /api/v1/hitl/workflows/{workflow_id}
       - Get workflow details

POST   /api/v1/hitl/workflows/{workflow_id}/execute
       - Execute workflow for checkpoint
       Request: {checkpoint_id, context}

GET    /api/v1/hitl/workflows/instances/{instance_id}
       - Get workflow instance status


# ==================== Escalation Management ====================

GET    /api/v1/hitl/escalations
       - List escalation cases
       Query: ?project_id=&status=&escalated_to_role=

GET    /api/v1/hitl/escalations/{escalation_id}
       - Get escalation details

POST   /api/v1/hitl/escalations/{escalation_id}/resolve
       - Resolve escalation
       Request: {resolution, resolution_details}

GET    /api/v1/hitl/audit/events
       - Get audit events
       Query: ?project_id=&start_date=&end_date=&event_type=
