================================================================================
【 STEP 1 】 프로젝트 생성 (Project Creation)
================================================================================
API 엔드포인트: POST /api/v1/projects
요청 모델: CreateProjectRequest

본 단계에서는 고객사의 기본 정보와 현황을 입력하여 새로운 컨설팅 프로젝트를
생성한다. 이 정보는 이후 모든 분석의 기초 데이터로 활용된다.

--------------------------------------------------------------------------------
1.1 기업 기본 정보 (Company Basic Information)
--------------------------------------------------------------------------------

[필드명]                   [데이터 타입]      [필수]   [설명]
─────────────────────────────────────────────────────────────────────────────────
name                      String            O       기업명을 입력한다. 공식 법인명
                                                    또는 통용되는 상호명을 기재한다.

industry                  Enum              O       산업 분류를 선택한다.
                                                    - manufacturing: 제조업
                                                    - finance: 금융업
                                                    - healthcare: 헬스케어
                                                    - retail: 유통/소매
                                                    - logistics: 물류
                                                    - energy: 에너지
                                                    - telecom: 통신
                                                    - public: 공공
                                                    - other: 기타

company_size              Enum              O       기업 규모를 선택한다.
                                                    - startup: 스타트업 (50인 미만)
                                                    - small: 중소기업 (50-300인)
                                                    - medium: 중견기업 (300-1000인)
                                                    - large: 대기업 (1000인 이상)
                                                    - enterprise: 대규모 기업집단

business_description      String            O       사업 개요를 서술한다. 주요 사업
                                                    영역, 핵심 제품/서비스, 시장
                                                    포지션 등을 포함하여 기술한다.
─────────────────────────────────────────────────────────────────────────────────

--------------------------------------------------------------------------------
1.2 IT 인프라 정보 (IT Infrastructure)
--------------------------------------------------------------------------------

[필드명]                   [데이터 타입]      [필수]   [설명]
─────────────────────────────────────────────────────────────────────────────────
has_cloud                 Boolean           O       클라우드 환경 보유 여부를
                                                    입력한다. 퍼블릭, 프라이빗,
                                                    하이브리드 클라우드를 포함한다.

cloud_provider            String            -       클라우드 제공자를 입력한다.
                                                    (예: AWS, GCP, Azure, NCP 등)
                                                    has_cloud가 true인 경우 기재한다.

has_data_warehouse        Boolean           O       데이터 웨어하우스 보유 여부를
                                                    입력한다. 구조화된 분석용
                                                    데이터 저장소의 존재 여부이다.

has_data_lake             Boolean           O       데이터 레이크 보유 여부를
                                                    입력한다. 비정형 데이터를
                                                    포함한 대용량 저장소이다.

server_count              Integer           O       운영 중인 서버 수를 입력한다.
                                                    물리 서버와 가상 서버를
                                                    모두 포함한 총 수이다.

gpu_available             Boolean           O       GPU 서버 보유 여부를 입력한다.
                                                    AI/ML 학습용 GPU 자원의
                                                    가용 여부를 확인한다.

legacy_system_count       Integer           O       레거시 시스템 수를 입력한다.
                                                    현대화가 필요한 구형 시스템의
                                                    수를 파악하여 통합 복잡도를
                                                    평가하는 데 활용한다.

api_capability            Boolean           O       API 연동 가능 여부를 입력한다.
                                                    기존 시스템과의 연계를 위한
                                                    API 인프라 존재 여부이다.

security_level            String            O       보안 수준을 선택한다.
                                                    - basic: 기본 보안
                                                    - standard: 표준 보안 체계
                                                    - high: 고수준 보안 (금융권 등)
─────────────────────────────────────────────────────────────────────────────────

--------------------------------------------------------------------------------
1.3 데이터 자산 정보 (Data Assets)
--------------------------------------------------------------------------------

[필드명]                   [데이터 타입]      [필수]   [설명]
─────────────────────────────────────────────────────────────────────────────────
data_volume_tb            Float             O       총 데이터 보유량을 TB 단위로
                                                    입력한다. 정형, 비정형 데이터를
                                                    모두 포함한 전체 규모이다.

structured_ratio          Float             O       정형 데이터 비율을 0~1 사이의
                                                    값으로 입력한다.
                                                    (예: 0.7 = 70%가 정형 데이터)

data_quality_score        Float             O       데이터 품질 점수를 1~5점
                                                    척도로 입력한다.
                                                    - 1: 매우 낮음 (일관성 없음)
                                                    - 2: 낮음 (부분적 품질 관리)
                                                    - 3: 보통 (기본 품질 관리)
                                                    - 4: 높음 (체계적 관리)
                                                    - 5: 매우 높음 (자동화된 관리)

has_data_governance       Boolean           O       데이터 거버넌스 체계 보유
                                                    여부를 입력한다. 데이터 정책,
                                                    표준, 품질 관리 프로세스의
                                                    존재 여부를 확인한다.

data_sources              List[String]      O       데이터 소스 목록을 입력한다.
                                                    (예: ERP, CRM, IoT센서,
                                                    웹로그, 외부API 등)

historical_data_years     Integer           O       축적된 히스토리 데이터 기간을
                                                    연 단위로 입력한다. ML 모델
                                                    학습에 활용 가능한 과거 데이터
                                                    축적 기간이다.
─────────────────────────────────────────────────────────────────────────────────

================================================================================
【 STEP 2 】 Stage 1: AI 비전 및 전략 수립 (Strategy)
================================================================================

Stage 1은 고객사의 AI 성숙도를 진단하고, AI 적용 기회를 발굴하며,
중장기 AI 로드맵을 수립하는 전략 단계이다.

--------------------------------------------------------------------------------
2.1 AI 성숙도 진단 (Maturity Assessment)
--------------------------------------------------------------------------------
API 엔드포인트: POST /api/v1/framework/projects/{project_id}/stage1/maturity-assessment
요청 모델: MaturityAssessmentInput

4대 영역에 대해 각각 4개 항목을 1~5점 척도로 평가한다.

--------------------------------------------------------------------------------
2.2 AI 기회 발굴 (Opportunity Discovery)
--------------------------------------------------------------------------------
API 엔드포인트: POST /api/v1/framework/projects/{project_id}/stage1/opportunities
요청 모델: OpportunityInput

각 AI 적용 기회에 대해 아래 항목을 입력한다. 일반적으로 5~10개의 기회를
발굴하여 입력한다.

- 성숙도 진단 (Maturity Assessment): 조직의 AI 역량을 4대 영역(전략, 조직,
  기술, 프로세스)에서 1~5점으로 평가하는 진단 활동이다.

- Quick Win: 높은 효과와 쉬운 구현이 가능한 AI 적용 기회로, 초기 성과
  창출에 적합한 과제이다.

- MLOps: 머신러닝 모델의 개발, 배포, 운영을 체계적으로 관리하기 위한
  실천 방법론 및 도구 체계이다.

- Feature Store: ML 모델 학습 및 서빙에 사용되는 특성(Feature)을 중앙에서
  관리하고 재사용할 수 있게 하는 저장소이다.

- XAI (Explainable AI): AI 모델의 의사결정 과정을 인간이 이해할 수 있도록
  설명하는 기술이다.

- 드리프트 (Drift): 모델 학습 시점과 운영 시점의 데이터 분포가 변화하여
  모델 성능이 저하되는 현상이다.

- Canary 배포: 새 버전을 소규모 트래픽에만 먼저 적용하여 문제를 조기에
  발견하는 배포 전략이다.

- CoE (Center of Excellence): AI 전문 역량을 집중시킨 전담 조직으로,
  전사 AI 프로젝트를 지원하고 표준을 수립한다.

- ISO/IEC 38500: IT 거버넌스의 국제 표준으로, EDM(평가-지휘-모니터링)
  사이클과 6대 원칙(책임, 전략, 획득, 성과, 적합성, 인간행동)을 정의한다.

- ISO/IEC 24030: AI 시스템 역량 평가를 위한 국제 표준으로, 조직의 AI
  준비도와 성숙도를 5대 영역에서 진단하는 프레임워크를 제공한다.

- ISO/IEC 42001: AI 관리 시스템(AIMS)의 국제 표준으로, 조직이 AI 시스템을
  책임감 있게 개발, 배포, 운영하기 위한 요구사항을 정의한다.

- ISO/IEC 23053: ML 프레임워크 국제 표준으로, 머신러닝 시스템의 구성요소와
  파이프라인 아키텍처를 정의한다.

- EDM 사이클 (Evaluate-Direct-Monitor): ISO 38500에서 정의한 IT 거버넌스
  핵심 활동 사이클로, 평가-지휘-모니터링의 지속적인 순환을 의미한다.

- AIMS (AI Management System): ISO 42001에서 정의한 AI 관리 시스템으로,
  조직이 AI를 체계적으로 관리하기 위한 정책, 프로세스, 통제를 포함한다.

- RACI 매트릭스: 역할과 책임을 정의하는 도구로, Responsible(책임자),
  Accountable(최종 결정자), Consulted(협의자), Informed(통보 대상)를 명시한다.

- 세션 저장 (Session Save): 컨설팅 작업 중 입력한 데이터를 브라우저의
  localStorage에 저장하여 작업 중단 후에도 데이터를 유지할 수 있게 하는
  기능이다. 섹션별로 저장 및 복원이 가능하다.

                      【 세션 저장/불러오기 기능 】
================================================================================

본 플랫폼은 컨설팅 작업의 연속성을 보장하기 위해 섹션별 세션 저장/불러오기
기능을 제공한다.

┌─────────────────────────────────────────────────────────────────────────────┐
│                          세션 저장 기능                                      │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                              │
│  1. 자동 저장                                                               │
│     - 입력 필드 변경 시 자동으로 localStorage에 저장                        │
│     - 페이지 이동 시 자동 저장                                              │
│     - 브라우저 탭 전환 시 자동 저장                                         │
│                                                                              │
│  2. 수동 저장                                                               │
│     - 사이드바 하단의 '저장' 버튼 클릭                                      │
│     - 현재 활성 페이지의 모든 입력 데이터 저장                              │
│                                                                              │
│  3. 불러오기                                                                │
│     - 사이드바 하단의 '불러오기' 버튼 클릭                                  │
│     - 저장된 데이터가 있으면 자동으로 폼에 복원                             │
│     - 저장 시간 표시로 데이터 최신성 확인 가능                              │
│                                                                              │
│  4. 내보내기/가져오기                                                       │
│     - 전체 세션 데이터를 JSON 파일로 내보내기                               │
│     - 다른 브라우저나 기기에서 JSON 파일로 가져오기                         │
│     - 백업 및 복원 용도로 활용                                              │
│                                                                              │
│  5. 저장 상태 표시                                                          │
│     - 각 메뉴 항목 옆에 저장 상태 표시 (녹색: 저장됨)                       │
│     - 섹션 헤더에 진행률 표시 (저장된 페이지 수 / 전체 페이지 수)          │
│                                                                              │
└─────────────────────────────────────────────────────────────────────────────┘
