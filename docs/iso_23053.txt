ISO/IEC 23053:2022 - 기계 학습을 사용하는 AI 시스템 프레임워크
Framework for AI Systems Using Machine Learning - 구현 가이드
================================================================================

1. 개요 (Overview)
================================================================================

1.1 표준 소개
-----------------------------------------------------------------------------
ISO/IEC 23053:2022는 기계 학습(Machine Learning, ML)을 사용하는 인공지능(AI)
시스템의 프레임워크를 정의하는 국제 표준입니다. 이 표준은 ML 기반 AI 시스템의
구성요소, 기능적 구조, 그리고 시스템 간의 상호작용을 체계적으로 설명합니다.

1.2 표준의 목적
-----------------------------------------------------------------------------
- ML 기반 AI 시스템의 공통 프레임워크 제공
- 시스템 구성요소와 기능의 명확한 정의
- AI 시스템 개발/운영을 위한 참조 아키텍처 제공
- 이해관계자 간 공통 언어 확립
- 상호운용성 및 호환성 촉진

1.3 적용 범위
-----------------------------------------------------------------------------
- ML 알고리즘을 활용하는 모든 AI 시스템
- 지도/비지도/강화 학습 시스템
- 단일 모델 및 앙상블 시스템
- 온프레미스 및 클라우드 기반 시스템
- 배치 및 실시간 추론 시스템

│  │                 │ • 가중치 감쇠                                       │   │
│  ├─────────────────┼───────────────────────────────────────────────────┤   │
│  │ 체크포인트      │ • 주기적 모델 저장                                  │   │
│  │                 │ • 최적 모델 저장                                    │   │
│  │                 │ • 훈련 재개 지원                                    │   │
│  ├─────────────────┼───────────────────────────────────────────────────┤   │

│     • 동일 모델을 여러 GPU에 복제                                           │
│     • 데이터를 분할하여 각 GPU에서 처리                                     │

│     • 메모리 제약 해결                                                      │
│                                                                             │
│  3. 하이브리드 병렬 (Hybrid Parallelism)                                    │
│     • 데이터 + 모델 병렬 조합                                               │
│     • 대규모 훈련에 적합                                                    │

│  │                                                                       │   │
│  │    [구현 패턴]                                                         │   │

│                                                                             │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │ 4. 엣지 추론 (Edge Inference)                                         │   │

│  │    • 경량 모델 필요                                                    │   │
│  │    • 사용 사례: 모바일 앱, IoT 디바이스, 자동차                        │   │
│  │                                                                       │   │
│  │    [최적화 기법]                                                       │   │

│  │    • 지식 증류 (Knowledge Distillation)                               │   │
│  │    • 모델 압축                                                         │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
│  [모델 서빙 인프라]                                                          │

7.1 용어 정의
-----------------------------------------------------------------------------
• 특성 (Feature): 모델 입력으로 사용되는 데이터 속성
• 특성 스토어 (Feature Store): 특성의 중앙 저장소
• 모델 레지스트리 (Model Registry): 모델 버전 관리 저장소
• 드리프트 (Drift): 시간에 따른 데이터/모델 성능 변화
• 추론 (Inference): 학습된 모델로 예측 수행
• MLOps: ML 시스템의 운영 자동화 실천 방법
