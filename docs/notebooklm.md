### [슬라이드 2] Executive Summary

**제목**: 왜 AX(AI Transformation)인가?

**핵심 메시지**:
- 대한민국 산업 현장의 **80% 이상의 업무**가 아직 수기·엑셀·경험에 의존하고 있다
- AI를 직접 개발하는 엔지니어가 아니라, **자기 업무에 AI를 접목하는 실무 전문가 10만 명**을 양성하는 것이 목표이다
- "사람을 대체하는 것"이 아니라, **"사람이 더 가치 있는 일에 집중할 수 있도록"** 반복적 업무를 AI에게 맡기는 것이다

**핵심 수치**:
| 항목 | 목표 |
|------|------|
| AX 전문인력 양성 | **100,000명** |
| 대상 산업 도메인 | **7개** (제조, 금융, 공공, 물류, 의료, 교육, 국방) |
| 대상 부서 | **10개** (경영, 생산, 품질, 자재, 영업, R&D, 재무/인사, 수출, 고객지원 등) |
| 인증 체계 | **5등급** (Beginner → Master) |

### [슬라이드 6] 7개 멀티 에이전트 프레임워크 — 개요

**제목**: 7 Multi-Agent Frameworks — Overview

```
                    [100K-AX Expert Orchestrator]
                              │
        ┌───────┬───────┬─────┴─────┬───────┬──────────┬───────────┐
        │       │       │           │       │          │           │
   LangGraph  CrewAI  AutoGen    DSPy  LangChain  LlamaIndex  Native
   StateGraph  Role   GroupChat  Sig/   LCEL       RAG        Sequential
   Workflow   Based   Dialog    CoT   Pipeline   Workflow   Orchestrator
```

**왜 7개인가?**:
- 단일 프레임워크는 모든 상황에 최적이 아니다
- 7개를 **동일 태스크**에 적용하여 결과를 비교·선택할 수 있다
- 고객 상황(데이터 유형, 팀 역량, 예산)에 맞는 최적 프레임워크를 추천

> 디자인 지시: 트리 구조 다이어그램. 상단 오케스트레이터에서 7개 리프로 분기.

### [슬라이드 8] DX-to-AX 전환 개념

**제목**: DX-to-AX: 디지털 전환에서 AI 전환으로

**5단계 성숙도 레벨**:

| Level | 명칭 | 설명 | 예시 (품질 검사) |
|:-----:|------|------|----------------|
| **0** | Manual | 수기·경험 의존 | 육안 검사 + 종이 기록 |
| **1** | Digital | 전산화·시스템 도입 | ERP 입력 + 엑셀 분석 |
| **2** | Data-Driven | 데이터 기반 의사결정 | 불량 패턴 데이터 분석 |
| **3** | AI-Augmented | AI가 사람을 보조 | AI 비전 카메라 자동 판별 + 사람 확인 |
| **4** | Autonomous | AI 자율 운영 | AI 판별 + 공정 조건 자동 조정 |

**핵심 메시지**: AX는 "Level 0~1에서 Level 3~4로의 전환"이다. AI 전문가가 아니라 **업무 전문가가 주도**한다.

> 디자인 지시: 좌에서 우로 흐르는 화살표형 프로세스 다이어그램. 각 레벨을 색상으로 구분 (회색→청색→녹색→보라색→골드).

### [슬라이드 9] DX-to-AX 전환 — 실무 적용 예시

**제목**: DX-to-AX 전환 실무 적용 (4개 업무 영역)

| 업무 영역 | Level 0 (수기) | Level 2 (데이터) | Level 4 (자율) |
|----------|---------------|-----------------|---------------|
| **생산 계획** | 경험 기반 수기 계획 | 엑셀 분석 + 과거 데이터 참조 | AI 자동 스케줄링 + 실시간 조정 |
| **재고 관리** | 담당자 감 + 전화 확인 | ERP 재고 현황 조회 | AI 수요 예측 → 자동 발주 |
| **고객 상담** | 전화 + 수기 메모 | CRM 이력 조회 | AI 챗봇 자동 응대 + 사람 에스컬레이션 |
| **문서 검토** | 수작업 전수 검토 | 키워드 검색 + 필터링 | AI 자동 분류 + 이상 항목 자동 플래깅 |

**각 전환에서 공통적으로 필요한 것**: ① 데이터 수집 체계 ② 업무 프로세스 표준화 ③ 현장 실무자의 AX 역량

> 디자인 지시: 4행 3열 매트릭스. 각 셀에 아이콘 + 핵심 키워드. Level 0→2→4 화살표 표시.

### [슬라이드 17] AX 기회 발굴 — 자동 발굴 조건과 점수 체계

**제목**: AX Discovery — 발굴 조건 & Scoring

**자동 발굴 조건**:

| 조건 | 발굴 유형 | 목표 레벨 |
|------|----------|:--------:|
| 현재 수행 방법 = 수기/엑셀 | 자동화 기회 | Level 2 |
| 오류율 > 3% | AI 품질 개선 | Level 3 |
| 인력 3명+ & 데이터 보유 | AI 의사결정 지원 | Level 3 |
| 데이터량 medium 이상 | 예측 분석 | Level 3 |

**4차원 점수 산출 알고리즘**:

| 차원 | 가중치 | 산출 기준 |
|------|:------:|----------|
| ROI 잠재력 | **30%** | 연간 비용 × 인력 수 × 오류율 |
| 구현 용이성 | **25%** | 복잡도 역수 × 기존 시스템 활용도 |
| 데이터 준비도 | **25%** | 데이터 보유 여부 × 데이터 양 |
| 조직 준비도 | **20%** | 디지털 수준 × 부서 변화 수용도 |

**종합 점수** = Σ (각 차원 점수 × 가중치) → 100점 만점

> 디자인 지시: 상단에 조건 테이블, 하단에 4차원 가중치를 원형 차트로 시각화.
