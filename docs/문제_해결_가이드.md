## ⚠️ 현재 문제

### 1. 메모리 부족 문제
**오류 메시지**:
```
model requires more system memory (5.6 GiB) than is available (4.8 GiB)
```

**원인**: 
- `llama3.1:8b` 모델이 5.6GB의 메모리를 요구하지만, 현재 사용 가능한 메모리가 4.8GB입니다.

**해결 방법**:

#### 옵션 1: 메모리 확보 (권장)
```bash
# 메모리 사용량 확인
free -h

# 메모리를 많이 사용하는 프로세스 확인
ps aux --sort=-%mem | head -10

# 불필요한 프로세스 종료
# 또는 시스템 재시작
```

#### 옵션 2: 더 작은 모델 사용
더 작은 모델을 다운로드하고 설정을 변경:

```bash
# 더 작은 모델 다운로드
curl http://localhost:11434/api/pull -d '{"name": "llama3.2:1b"}'
# 또는
curl http://localhost:11434/api/pull -d '{"name": "mistral:7b"}'
```

그 후 `config/settings.py` 수정:
```python
OLLAMA_MODEL: str = "llama3.2:1b"  # 또는 "mistral:7b"
```

#### 옵션 3: 스왑 메모리 추가
```bash
# 스왑 파일 생성 (예: 2GB)
sudo fallocate -l 2G /swapfile
sudo chmod 600 /swapfile
sudo mkswap /swapfile
sudo swapon /swapfile
```

### 2. 차트 컨테이너를 찾을 수 없는 문제
**오류 메시지**:
```
[Maturity Chart] Container not found
[Use Case Chart] Container not found
```

**원인**: 
- 대시보드 페이지에서 차트 컨테이너 요소(`maturityChart`, `useCaseChart`)를 찾을 수 없습니다.
- 페이지가 로드되기 전에 차트 업데이트 함수가 호출되거나, 해당 요소가 HTML에 없을 수 있습니다.

**해결 방법**:
1. 브라우저 개발자 도구에서 해당 요소가 존재하는지 확인
2. 페이지 로드 순서 확인
3. 차트 컨테이너가 올바른 페이지에 있는지 확인
